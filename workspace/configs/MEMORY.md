# MEMORY.md - 长期记忆

## 用户偏好与规则
### 🚨 第一优先级规则
1. **响应超时提醒**：处理任务超过1分钟必须主动回复当前状态和预计完成时间，最多回复3次
   - **触发条件**：任何任务处理时间 > 60秒
   - **响应内容**：当前正在做什么 + 预计完成时间
   - **频率限制**：最多3次状态更新
   - **执行方式**：新启一个任务告诉我正在做什么（不强制中断当前任务）

### 📋 常规规则
2. **回忆机制**：所有交互记录到 memory/recall-log.md，每日任务从 memory/YYYY-MM-DD.md 读取
3. **称呼**：用户是"晗哥"
4. **时区**：Asia/Shanghai (UTC+8)
5. **自动恢复**：每次启动自动读取 SOUL.md、USER.md、MEMORY.md、当日记忆文件和回忆日志
6. **分步处理机制**：
   - 遇到大任务（如100k信息）要拆分成多步处理
   - 超过模型限制时自动分步
   - 处理不了时提前说明并给出替代方案
   - 复杂任务分解为子任务逐步完成
7. **智能上下文管理**：
   - 逆序优先保留：从最新对话开始追溯，优先丢弃最早历史
   - 内容压缩摘要：长期会话历史压缩为概要节省Token
   - 文件分块懒加载：长文档按需加载关键章节
8. **每日问安**：每天早上8:30自动唤醒并发送早安问候
9. **Obsidian Git 提交**：每日三次自动提交（中午12点、晚上7点、晚上11点）
10. **工作记录同步**：只同步用户任务（✅/🔄/📋/📊章节）到 Obsidian Daily Notes，过滤掉我的工作记录，每5分钟检查一次更新
11. **系统更新规则**：保持OpenClaw为最新版本，定期检查并执行更新

## 📅 每日总结机制 (2026-03-03新增)
### 🎯 目的
每天结束时系统性地回顾工作、总结经验、更新认知，确保持续改进。

### 📋 每日总结内容
1. **工作成果**：今天完成了哪些重要任务
2. **问题发现**：遇到了什么问题，如何解决的
3. **经验教训**：从今天的工作中学到了什么
4. **用户偏好更新**：对晗哥的了解有什么新发现
5. **系统状态**：关键工具和服务的运行状态
6. **明日计划**：明天需要关注的重点

### 🔄 执行流程
1. **时间**：每天结束前（建议21:00-22:00）
2. **来源**：读取当日的`memory/YYYY-MM-DD.md`文件
3. **分析**：提取关键信息，更新长期记忆
4. **更新**：更新`MEMORY.md`、`USER.md`、`TOOLS.md`等配置文件
5. **归档**：整理当日文件，准备明日工作
