# openclaw.json 安全管理规范

## 核心原则
- 皮休拥有一票决定权
- 未经同意不得擅自修改
- 自动恢复保障业务连续性

---

## 1. 每日备份

- **执行时间**：每晚 12:00
- **备份内容**：完整备份 `~/.openclaw/openclaw.json`
- **备份位置**：`~/.openclaw/backups/openclaw-YYYY-MM-DD.json`
- **保留数量**：保留最近30天

```bash
# 备份命令
mkdir -p ~/.openclaw/backups
cp ~/.openclaw/openclaw.json ~/.openclaw/backups/openclaw-$(date +%Y-%m-%d).json
```

---

## 2. 修改前备份

- **触发时机**：任何修改 openclaw.json 之前
- **备份命名**：`openclaw-YYYY-MM-DD-HHMMSS-修改前.json`
- **保留**：永久保留

---

## 3. 修改申请流程

### 必须申请的修改
- 任何对 openclaw.json 的修改
- 任何配置变更
- 任何可能导致系统变动的操作

### 申请格式
```
📋 修改申请 - {小皮名称}

- 申请人：xxx
- 修改内容：xxx
- 修改原因：xxx
- 预期影响：xxx

请皮休回复「同意」后执行
```

### 规则
- ❌ 回复"好的"、"可以"、"嗯"等 ≠ 同意
- ❌ 回复表情 ≠ 同意
- ✅ 仅「同意」二字 = 执行许可
- 其他任何回复 = 不执行

---

## 4. Gateway 重启申请

### 申请格式
```
📋 Gateway重启申请 - {小皮名称}

- 申请人：xxx
- 重启原因：xxx
- 更新内容：xxx
- 等待时间：5秒后自动重启

请皮休回复「同意」后执行
```

### 重启后必须汇报
1. 重启是否成功
2. 正在进行的任务
3. 是否有未完成的工作
4. 提醒皮休需要关注的事项

---

## 5. 意外中断自动恢复

### 监测场景
- 断网
- 断连
- 服务异常
- 意外终止

### 自动恢复流程
1. 检测到异常 → 等待网络恢复
2. 自动重连 Gateway
3. 自动重启服务（如需要）
4. **第一时间汇报皮休**：
   - 发生了什么
   - 持续多久
   - 是否有数据丢失
   - 当前状态

---

## 6. 重启后必做

### 第一时间
1. 读取 memory/ 查看之前的工作记录
2. 回忆正在进行的任务
3. 汇报皮休：是否有未完成的工作

### 汇报格式
```
📋 重启后状态汇报 - {小皮名称}

✅ 已恢复
- 重启原因：xxx
- 持续时间：xxx

📝 进行中的任务：
1. xxx - 状态：xxx

⚠️ 需要关注：
- xxx
```

---

## 7. 问题反思机制

### 每次问题后必须反思
1. **发生了什么**：详细描述问题
2. **为什么发生**：分析根因
3. **如何避免**：制定预防措施
4. **执行情况**：是否已落实

### 反复出现的问题
- 第一次：分析原因，制定方案
- 第二次：审查方案是否有效
- 第三次：彻底解决问题，更换方法

### 禁止
- ❌ 同样的问题出现第二次
- ❌ 同样的问题出现第三次
- ❌ 不记录、不反思、不改进

---

## 执行检查清单

每次操作前检查：
- [ ] 是否需要申请？
- [ ] 是否已备份？
- [ ] 是否已获得「同意」？
- [ ] 是否记录到 memory？
- [ ] 重启后是否汇报？
