# 管家工作流程讨论汇总

**日期**：2026-03-19
**汇总人**：小皮管家
**参与**：产品、设计、研发、测试、部署、审核小皮

---

## 一、各方意见汇总

### 1. 发现的问题

| # | 问题 | 提出者 | 影响 |
|---|------|--------|------|
| 1 | 缺少任务优先级判断 | 产品、研发 | 紧急任务可能被耽误 |
| 2 | 缺少验收标准确认 | 产品、测试 | 执行中可能出现返工 |
| 3 | 多小皮协作场景缺失 | 产品、测试 | 协作任务无法流转 |
| 4 | 进度同步不及时 | 产品、审核 | 皮休不知道执行进展 |
| 5 | 缺少技术可行性评估 | 研发 | 技术风险未提前识别 |
| 6 | 缺少审核节点 | 审核、测试 | L2+任务缺少质量把关 |
| 7 | 失败处理不够细化 | 研发、测试、部署 | 没有区分可重试/不可重试 |
| 8 | 步骤编号跳跃(11→13) | 研发 | 流程不完整 |
| 9 | 缺少超时机制 | 研发、审核 | 可能无限等待 |
| 10 | 缺少环境确认 | 部署 | 部署环境风险未评估 |
| 11 | 规范标准定义模糊 | 审核 | 符合规范无具体标准 |

### 2. 遗漏的环节

| 遗漏环节 | 提出者 | 建议添加位置 |
|----------|--------|--------------|
| 任务优先级 | 产品、研发 | 第4步后 |
| 预估时间 | 研发 | 第8步前 |
| 验收标准确认 | 产品、测试 | 第4-5步间 |
| 环境与权限确认 | 部署 | 第4步后（部署任务） |
| 部署风险评估 | 部署 | 第8步前（部署任务） |
| 备份确认 | 部署 | 执行前（涉及删除/修改） |
| 审核节点 | 审核、测试 | 第6步后（L2+任务） |
| 质量验证 | 测试 | 第13步前（L3+任务） |
| 通知相关小皮 | 部署 | 任务完成后 |
| 每日定时任务 | 审核 | 区分事件触发vs定时触发 |
| 任务队列管理 | 测试 | 多任务并行处理 |

### 3. 各角色专项建议

**产品小皮**：
- 增加任务卡片（名称、负责人、验收标准、优先级、协作小皮）
- 简化的标准流程：接收→排序→确认标准→执行→完成→归档

**研发小皮**：
- 增加技术可行性评估
- 区分可重试失败和不可重试失败
- 复杂任务增加拆解步骤

**测试小皮**：
- L3+任务增加测试验证节点
- 失败必须记录原因到memory

**部署小皮**：
- 增加环境确认和风险评估节点
- 部署操作分级（1-4级）
- 部署完成后通知相关小皮

**审核小皮**：
- L2+任务增加审核节点
- 配套规范清单（checklist）
- 每步设置时间上限，超时自动升级

## 三、配套规范建议

### 1. 任务卡片标准

```
任务卡片：
- 任务名称：
- 负责人：
- 任务类型：□问答 □协作 □系统
- 优先级：□P0 □P1 □P2 □P3
- 验收标准：
- 预计时间：
- 协作小皮：
- 审核要求：□L1 □L2 □L3 □L4
- 当前状态：
```

### 2. 部署任务卡片

```
部署任务卡片：
- 部署环境：□测试 □预发布 □生产
- 预计停机时间：____分钟
- 回滚方案：□有 □无
- 备份确认：□已备份
- 监控告警：□已配置 □无需配置
- 部署级别：□1级 □2级 □3级 □4级
- 需要通知：□测试 □产品 □...
```

### 3. 审核分级标准

| 级别 | 触发条件 | 审核内容 | 时效 |
|------|----------|----------|------|
| L1 | 日常任务 | 自检即可 | - |
| L2 | 重要变更 | 对应专家审核 | ≤4h |
| L3 | 上线/新功能 | 审核小皮审核 | ≤24h |
| L4 | 重大决策 | 皮休决策 | 按需 |

### 4. 超时机制

| 节点 | 超时时间 | 超时动作 |
|------|----------|----------|
| 等待皮休回复 | 5分钟 | 提醒一次 |
| 任务执行 | 预计时间×1.5 | 主动汇报进度 |
| 审核节点 | 4小时 | 自动升级皮休 |
| L3任务总时限 | 24小时 | 提醒+汇总状态 |
