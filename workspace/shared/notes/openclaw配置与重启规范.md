# openclaw.json 备份与修改规范

**建立日期**: 2026-03-18  
**制定人**: 皮休  
**执行人**: 所有专家小皮

---

## 核心原则

> **任何修改前必须向皮休申请，只有回复"同意"才能执行**

---

## 1. 每日自动备份

- **执行时间**: 每晚 12:00
- **备份文件**: openclaw.json
- **备份位置**: /home/pixiu/.openclaw/backups/
- **命名格式**: openclaw.json.backup-YYYY-MM-DD

```bash
# 备份脚本示例
cp /home/pixiu/.openclaw/openclaw.json /home/pixiu/.openclaw/backups/openclaw.json.backup-$(date +%Y-%m-%d)
```

---

## 2. 修改前单独备份

- **触发时机**: 每次修改 openclaw.json 之前
- **备份命名**: openclaw.json.backup-YYYY-MM-DD-HHMMSS

---

## 3. 修改申请流程

### 必须遵守的流程
1. 向皮休说明要做什么更新
2. 等待皮休回复"同意"
3. 只有回复"同意"才执行
4. 回复其他任何信息都不修改

### 申请示例
```
皮休好，我需要对 openclaw.json 做以下更新：
[具体更新内容]
请确认是否同意？
```

---

## 4. Gateway 重启规范

### 重启申请流程
1. 向皮休说明做了哪些更新
2. 说明为什么要重启
3. 等待皮休回复"同意"
4. 皮休回复"同意"后，等待 5 秒自动重启

### 重要提醒
- 是**自动重启**，不是让皮休手动重启
- 等待 5 秒钟后自动执行
- 皮休回复其他任何信息都不重启

---

## 5. 意外中断自动恢复

### 监测与自动重启
- 断网断连等意外中断时
- 自动监测连接状态
- 自动尝试重新连接
- 自动重启 gateway

### 必须告知皮休
- 什么原因导致的重启
- 是否恢复成功
- 当前状态

---

## 6. 重启后第一时间汇报

### 重启完成后必须做
1. 立即回忆正在做的事
2. 检查是否有未完成的任务
3. 第一时间向皮休汇报

### 汇报内容
- 上次正在做什么
- 任务状态
- 是否需要继续

---

## 7. 反思与改进

### 每次问题后必须反思
1. **为什么会出现这个问题？**
2. **如何能够避免再次发生？**
3. **如果反复出现，为什么？**

### 改进措施
- 分析根因
- 制定预防方案
- 避免第二次、第三次出现
- 记录到 self-improving/

---
