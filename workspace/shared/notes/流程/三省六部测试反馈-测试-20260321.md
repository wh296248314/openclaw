## 🔴 严重问题（阻塞）

### 问题1：皮休任务看板始终为空

| 项目 | 说明 |
|------|------|
| **现象** | 任务看板 Tab 显示 0 个任务 |
| **根因** | API 方法不匹配：`/api/tasks-list` 服务端是 **POST**，前端 JS 用 **GET** |
| **位置** | 前端：`dist/tasks.js` 第5行 `fetch(API+'/tasks-list')`（无method，默认GET）；后端：`server.py` `do_POST()` 方法里处理 |
| **影响** | 皮休任务看板完全无法使用，创建/读取任务均失败 |
| **修复建议** | 方案A：改前端 JS `fetch(API+'/tasks-list')` 为 `fetch(API+'/tasks-list', {method:'POST'})`；方案B：改后端 `do_GET()` 添加 `tasks-list` 处理 |

**验证**：
```bash
# GET 返回 404
curl http://127.0.0.1:7891/api/tasks-list
# → 404 Not Found

# POST 返回空
curl -X POST http://127.0.0.1:7891/api/tasks-list
# → {"ok":true,"tasks":[]}
```

### 问题2：六部尚书全部失踪，仪表盘只有1个官员

| 项目 | 说明 |
|------|------|
| **现象** | live-status 只返回 1 个官员（尚书令/小皮管家），其他6个官员全部不显示 |
| **预期** | 应显示：尚书令(管家)、太子(产品)、侍中(审核)、礼部(设计)、工部(研发)、刑部(测试)、兵部(部署) |
| **实际** | 只有尚书令在线 🟢 |
| **根因** | live_status.json 数据来源不完整，只同步了 main agent |
| **影响** | 「军机处·三省六部总控台」名不副实，看不到团队全貌 |

**验证**：
```json
// curl /api/live-status → officials 数组只有1个
{
  "officials": [
    {
      "id": "main",
      "label": "尚书令",
      "role": "小皮管家",
      "model": "minimax/MiniMax-M2.5-highspeed",
      "heartbeat": {"status": "active", "label": "🟢 活跃 0分钟前"}
    }
  ]
}
```

**修复建议**：
1. 检查 `refresh_live_data.py` 脚本，确认是否只读取了 main 会话
2. 确认 `agent_config.json` 中的 agents 列表是否包含全部7个角色
3. `curl /api/agent-config` 只返回 5 个 agents，可能不完整
