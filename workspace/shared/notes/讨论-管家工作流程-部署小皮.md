# 讨论-管家工作流程-部署小皮

**讨论人**: 部署专家小皮
**时间**: 2026-03-19 14:20

---

## 一、流程有什么问题？

### 问题1：第8步"告知多久并执行"缺少前置评估

- 部署任务不同于普通任务，需要评估：
  - 预计停机时间窗口
  - 回滚方案和回滚时间
  - 监控告警配置
  - 数据一致性保障

**建议**：部署类任务在第8步之前增加"部署风险评估"节点

### 问题2：第5步"是否符合规范"对部署任务缺少具体标准

- 部署规范包括：
  - 生产环境操作需审批
  - 删除操作需备份
  - 重大变更需回滚方案
  - 监控告警必须配置

**建议**：为不同任务类型配置不同的规范checklist

### 问题3：缺少"环境确认"环节

- 第4步判定任务类型后，应该确认：
  - 目标环境（测试/预发布/生产）
  - 执行权限是否足够
  - 依赖服务状态

**建议**：第4步后增加"环境与权限确认"

### 问题4："失败"处理不够细化

- 部署失败可能是：
  - 部署失败（可重试）
  - 回滚失败（严重）
  - 监控告警（需要通知）

**建议**：失败类型分类处理

---

## 二、有没有遗漏环节？

### 遗漏1：备份确认

- 涉及配置文件修改、删除操作前
- 必须确认已备份

### 遗漏2：监控告警联动

- 部署任务完成后
- 需要确认监控已配置、告警已就绪

### 遗漏3：任务完成后通知相关小皮

- 部署完成后需要通知测试小皮验证
- 通知产品小皮功能已上线

### 遗漏4：GitHub提交前需要检查

- 部署任务可能涉及配置文件更新
- 应该确认哪些文件需要提交

## 三、如何更好地执行？

### 建议1：部署任务专用流程

```
接收任务 → 环境确认 → 风险评估 → 备份确认 
→ 部署执行 → 监控验证 → 结果反馈 → 通知相关小皮
```

### 建议2：部署任务卡片

```
部署任务卡片：
- 部署环境：□测试 □预发布 □生产
- 预计停机时间：____分钟
- 回滚方案：□有 □无
- 备份确认：□已备份
- 监控告警：□已配置 □无需配置
- 需要通知：□测试 □产品 □...
```

### 建议3：部署操作分级

| 级别 | 操作类型 | 要求 |
|------|----------|------|
| 1级 | 配置文件修改 | 备份+监控 |
| 2级 | 服务重启 | 备份+监控+回滚方案 |
| 3级 | 生产环境部署 | 全部+审批+通知 |
| 4级 | 删除/不可逆操作 | 冷静期+全程记录 |

### 建议4：与其他小皮的协作机制

- **部署前**：通知测试小皮准备验收
- **部署后**：告知相关小皮任务完成
- **回滚时**：立即通知所有相关方

---

## 四、结合三省六部审核机制

如果引入审核机制，部署任务可以这样设计：

1. L1级：配置文件修改 → 直接执行
2. L2级：普通服务部署 → 审核小皮确认
3. L3级：生产环境变更 → 门下省审核 + 工部执行
4. L4级：重大不可逆操作 → 皮休决策

---

## 五、总结

**核心建议**：
1. 增加"环境确认"和"风险评估"节点
2. 区分不同级别的部署操作
3. 部署完成后通知相关小皮
4. 回滚方案必须前置确认

**作为工部尚书，我愿意负责**：
- 制定部署类任务的规范checklist
- 维护部署操作分级标准
- 为其他小皮提供部署相关的咨询
