# 小皮团队重要对话同步机制

> **核心原则**：每次 `/new` 或 `/reset` 前，必须将重要结论同步到 memory 文件
> 创建时间：2026-03-20 | 制定者：小皮管家

---

## 一、同步内容

### 必须同步
- 皮休给的任何任务结论
- 各小皮协调的决定和结论
- 重要配置变更
- 待办事项和负责人

### 格式要求
```markdown
## [日期] 对话同步

### 皮休决策
- xxx

### 任务结论
- xxx

### 待办
- [ ] xxx（负责人：xxx）
```

---

## 二、执行时机

| 场景 | 操作 |
|------|------|
| 收到皮休明确任务 | 任务完成后立即同步 |
| 各小皮协调完成 | 结论达成后立即同步 |
| 每次 /new 或 /reset 前 | 必须执行一次 memory 更新 |
| 重要配置变更 | 变更后立即同步 |

---

## 三、各小皮操作步骤

### Step 1: 更新当前 session 的 memory
在 reset 前，运行以下命令将结论追加到 memory：

```bash
# 在各自 workspace 下执行
echo "## [日期] 对话同步

### 皮休决策
- xxx

### 任务结论
- xxx

### 待办
- [ ] xxx（负责人：xxx）" >> memory/YYYY-MM-DD.md
```

### Step 2: Git 同步
```bash
cd ~/.openclaw/workspace && git add -A && git commit -m "memory: 同步对话结论 - 日期"
```

---

## 四、检查机制

小皮管家每小时检查一次各小皮的 memory 变更，发现未同步的情况及时提醒。
