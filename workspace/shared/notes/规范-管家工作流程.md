## 快速概览（15节点）

```
【节点0】每日技能学习（9:30前）→ 检查+催促+汇报
        ↓
【节点1】接收任务 → 回忆规范+回复收到+判断优先级
        ↓
【节点1.5】规范不存在 → 自动补充+皮休确认
        ↓
【节点1.6】技能匹配 → 查找skills-index → 现成/组合/自建
        ↓
【节点2】判定类型 → 问答/协作/系统/部署
        ↓
【节点3】确认验收标准 → 完成定义+交付格式+时间
        ↓
【节点4】规范检查 → checklist对照+技术可行性
        ↓（不符合→告知原因→皮休确认）
【节点5】L2+审核 → 审核小皮（4h/24h/当天）
        ↓（驳回→修改→重新提交）
【节点6】告知结论 → 方案+时间+分工
        ↓
【节点7】等皮休回复
        ↓（不同意→结束）
【节点8】执行 → 使用技能+每50%时间反馈
        ↓（失败→记录+重新方案+皮休确认）
【节点9】质量验证（L2+）→ 测试/设计/审核小皮
        ↓（不通过→回节点8）
【节点10】反馈结果 → 结论先行+产出摘要
        ↓
【节点11】皮休验收
        ↓（返工→回节点8）
【节点12】记录今天工作 → memory
        ↓
【节点13】每日17:00提交GitHub
```

## 二、技能管理体系

### 1. 技能分类

| 类型 | 说明 | 例子 |
|------|------|------|
| **内置技能** | OpenClaw内置，可直接使用 | feishu-doc、weather、file-organizer |
| **自建技能** | 各小皮根据需求创建的技能 | kubectl速查、Semgrep |
| **组合技能** | 多个技能组合使用 | 自动备份+通知 |
| **待建技能** | 发现需求后评估创建 | 记录在skills-index.md |

### 2. 问题-技能映射表

| 皮休问题类型 | 对应技能 | 技能类型 | 状态 |
|--------------|----------|----------|------|
| 飞书文档相关 | feishu-doc | 内置 | ✅ |
| 飞书云盘相关 | feishu-drive | 内置 | ✅ |
| 天气查询 | weather | 内置 | ✅ |
| 文件整理 | file-organizer | 内置 | ✅ |
| K8s运维 | kubectl速查 | 自建 | ✅ |
| 代码安全审计 | Semgrep | 自建 | ✅ |
| 网页总结 | mh-summarize | 内置 | ✅ |

### 3. 技能匹配决策流程

```
遇到任务 → 分析需要什么能力
        ↓
查找 skills-index.md 映射表
        ↓
┌──────────────────────────────────────────────┐
│  是否有现成技能？                             │
│       ↓                                      │
│  有 → 直接调用 → 解决 → 记录使用经验           │
│       ↓ 无                                   │
│  可组合多个技能？ → 是 → 组合使用              │
│       ↓ 否                                   │
│  评估创建价值：                               │

│       ↓                                      │

## 三、任务分级标准

| 等级 | 说明 | 审核要求 | 时效要求 |
|------|------|----------|----------|
| P0 | 紧急/重大 | L3审核+皮休确认 | 立即处理，实时汇报 |
| P1 | 重要 | L2审核 | 4小时内完成 |
| P2 | 一般 | L1快速审核 | 当天完成 |

## 六、自动补充规范机制

### 触发条件
遇到无现有规范可遵循的场景时

### 执行流程
1. 分析任务需求，判断需要什么规范
2. 搜索相关规范
3. 制定临时规范
4. 记录到当日 memory
5. 告知皮休确认
6. 确认后 → 加入正式规范库

当前流程节点9提到"测试小皮验证"，但不够具体。建议补充：

- L2+任务完成后必须经过测试小皮验证

- 风险等级：高/中/低
- 长期影响：技术债务？

**评估结论**：
- ✅ 可行 → 继续执行
- ⚠️ 有风险 → 告知皮休
- ❌ 不可行 → 寻找替代方案
```
