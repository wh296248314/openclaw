## 二、技能管理体系

### 1. 技能分类

| 类型 | 说明 | 例子 |
|------|------|------|
| **内置技能** | 已有的可直接用技能 | feishu-doc、weather |
| **自建技能** | 各小皮自己创建的技能 | kubectl速查、Semgrep |
| **组合技能** | 多个技能组合使用 | 自动备份+通知 |
| **待建技能** | 发现需求后创建 | 根据问题创建 |

### 2. 问题-技能映射表

| 皮休问题类型 | 对应技能 | 技能类型 |
|--------------|----------|----------|
| 飞书文档相关 | feishu-doc | 内置 |
| 飞书云盘相关 | feishu-drive | 内置 |
| 代码安全审计 | Semgrep | 自建 |
| 文件整理 | file-organizer | 内置 |
| 天气查询 | weather | 内置 |
| K8s运维 | kubectl速查 | 自建 |
| Git操作 | git-essentials | 内置 |

### 3. 技能创建流程

```
遇到问题 → 查找技能映射表
     ↓
是否有技能？
     │
    是 → 直接调用 → 解决 → 记录使用经验
     │
    否 → 评估：
     │     · 频率高 → 创建自建技能
     │     · 偶尔 → 记录为待建技能
     │     · 可组合 → 创建组合技能
     ↓
创建/组合技能 → 纳入映射表 → 同步给各小皮
```

## 三、每日技能学习（节点0）

### 执行时间
- 每天 9:30 前完成检查

### 检查流程
1. 检查 `shared/notes/` 是否有当日技能分享
2. 如未提交，@all 催促
3. 9:45 仍未提交，向皮休汇报

### 学习要求
| 项目 | 要求 |
|------|------|
| 学习时间 | 每天9:30前完成 |
| 提交路径 | `shared/notes/技能分享-{小皮}-{YYYY-MM-DD}.md` |
| 内容要求 | 技能名称、特点(3点)、应用场景、学习资源 |
| 技能价值 | 分析是否可解决某类问题，是否纳入映射表 |

## 四、任务分级标准

| 等级 | 说明 | 审核要求 | 时效要求 |
|------|------|----------|----------|
| P0 | 紧急/重大 | L3审核+皮休确认 | 立即处理，实时汇报 |
| P1 | 重要 | L2审核 | 4小时内完成 |
| P2 | 一般 | L1快速审核 | 当天完成 |
| P3 | 日常 | 自动通过 | 尽快完成 |

---

## 五、审核时效标准

| 等级 | 时效 | 超时处理 |
|------|------|----------|
| L1 | 1小时内 | 自动通过 |
| L2 | 4小时内 | 提醒审核小皮 |
| L3 | 24小时内 | 自动升级至皮休 |
| L4 | 当天讨论 | 组织多方会审 |

## 六、执行过程反馈规则

| 预计时间 | 反馈时机 |
|----------|----------|
| <1分钟 | 执行完再反馈 |
| 1-5分钟 | 执行完反馈 |
| 5-30分钟 | 每50%时间反馈 |
| >30分钟 | 每15分钟反馈 |

## 八、自动补充规范机制

### 触发条件
当遇到无现有规范可遵循的场景时

### 执行流程
1. 分析任务需求，判断需要什么规范
2. 搜索相关规范
3. 制定临时规范
4. 记录到当日 memory
5. 告知皮休确认
6. 确认后 → 加入正式规范库
