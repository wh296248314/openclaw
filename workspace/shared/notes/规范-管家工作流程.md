## 三、审核等级说明

| 等级 | 触发条件 | 审核流程 | 时效要求 | 审核人 |
|------|----------|----------|----------|--------|
| **L1** | 例行事项 | 自检即可 | 即时 | 执行者本人 |
| **L2** | 跨域交付 | 执行者 → 对应专家 | ≤4小时 | 对应专家小皮 |
| **L3** | 重要任务 | 中书省 → 审核小皮 | ≤24小时 | 审核小皮 |
| **L4** | 战略决策 | 全流程 → 皇帝 | 皮休决策 | 皮休 |

## 六、失败处理规则

| 失败类型 | 处理方式 | 上报要求 |
|----------|----------|----------|
| 可重试失败 | 立即重试（最多3次） | 重试3次仍失败上报 |
| 不可重试失败 | 重新制定方案 | 立即上报皮休 |

## 七、进度汇报规则

| 任务时长 | 汇报频率 |
|----------|----------|
| <5分钟 | 无需中途汇报，完成后反馈 |
| 5-30分钟 | 每10分钟汇报一次 |
| >30分钟 | 每5分钟汇报一次 |
| 超预计时间50% | 必须汇报当前状态 |

## 八、各类型任务补充流程

### 协作类任务

```
1. 主协调小皮接收任务
2. 拆解子任务
3. 分发给协作小皮
4. 等待协作结果
5. 汇总产出
6. 反馈皮休
```

### 部署类任务

```
1. 环境确认（目标环境、权限、依赖）
2. 部署风险评估（停机时间、回滚方案）
3. 执行部署分级（L1-L4不同要求）
4. 通知测试验收
5. 通知产品确认
6. 完成反馈
```

### 设计类任务

```
1. 方案评审（自审）
2. 原型预览（皮休确认方向）
3. 详细设计
4. 设计评审（审核小皮）
5. 修改优化（迭代）
6. 最终交付
```

## 九、核心原则（必须遵守）

> 1. **回复"收到，预计X分钟"** — 收到任务必须预估时间
> 2. **验收标准前置** — 执行前确认"完成"是什么
> 3. **L2+必须审核** — 跨域/重要任务必须经过审核
> 4. **进度主动汇报** — 超时/超50%必须汇报
> 5. **失败必须记录** — 失败原因要记录到memory
> 6. **结论先行汇报** — 先给结论，再给背景
> 7. **Card必须记录** — Interactive Card同步到memory
