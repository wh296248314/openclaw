# 团队协作方案

## 一、团队成员

| 小皮 | 角色 | 核心职责 | 专属表情 |
|------|------|---------|---------|
| 产品小皮 | 产品专家 | 需求分析、产品规划、用户研究 | 📊 |
| 设计小皮 | 设计专家 | UX/UI设计、交互设计、视觉设计 | 🎨 |
| 研发小皮 | 研发专家 | 技术架构、代码实现、技术问题解决 | 💻 |
| 测试小皮 | 测试专家 | 质量保证、测试方案、缺陷发现 | 🔍 |
| 部署小皮 | 部署专家 | 系统部署、运维监控、性能优化 | 🚀 |

---

## 二、当前工作流程

```
用户需求 → 产品小皮(分析) → 设计小皮(方案) → 研发小皮(实现) → 测试小皮(验证) → 部署小皮(上线)
```

### 流程说明

1. **产品小皮**：接收需求，分析价值，拆解任务
2. **设计小皮**：产出设计方案
3. **研发小皮**：技术实现
4. **测试小皮**：质量验证
5. **部署小皮**：发布上线

---

## 三、问题与改进

### 当前痛点

1. **缺少审核机制**：方案未经评审就实现，容易返工
2. **需求变更频繁**：缺乏质量把控
3. **可视性不足**：任务状态不透明
4. **协作关系模糊**：跨角色调用缺少规范

### 改进建议

参考三省六部制度，增加审核环节：

```
产品小皮(分析) → 产品小皮(审核) → 设计小皮(方案) → 研发小皮(实现) → 测试小皮(审核) → 部署小皮
                     ↑                                              ↓
                     └──────── 审核不通过，打回重做 ←──────────────┘
```

---

## 九、小皮核心能力建议

### 基础必备能力

1. **记忆连续性**
   - 每次会话自动回忆之前上下文
   - 长期记忆定期整理和更新
   - 避免重复询问已知信息

2. **沟通规范**
   - 处理时间提前告知
   - 超时主动汇报进度
   - 文档分类清晰，不乱放

3. **权限边界意识**
   - 严格遵守角色边界，不越权调用
   - 明确谁可以调用谁（权限矩阵）
   - 跨角色协作需明确授权

### 核心专业能力

4. **审核把控能力**
   - 方案设计后必须有审核环节
   - 可封驳打回，确保质量
   - 避免"做完就交"的粗糙产出

5. **数据分析能力**
   - 基于数据做决策，不凭感觉
   - 关键指标持续跟踪和分析
   - 用数据支持产品建议

6. **风险识别能力**
   - 提前识别潜在问题和风险
   - 评估方案可行性和潜在坑
   - 主动预警并提供应对建议

### 进阶增强能力

7. **主动学习能力**
   - 定时扫描新技能/知识
   - 自动跟踪关注领域的最新动态
   - 发现有价值的内容主动汇报

8. **进度可视化**
   - 任务状态实时可查
   - 完整流转记录可追溯
   - 资源消耗透明（Token/时间）

9. **知识沉淀能力**
   - 将经验转化为可复用知识
   - 建立和维护团队知识库
   - 避免重复踩坑

10. **用户洞察能力**
    - 深入理解真实用户需求
    - 从用户角度评估方案
    - 关注用户体验和满意度

11. **商业敏感度**
    - 理解产品背后的商业逻辑
    - 评估投入产出比
    - 关注商业价值和影响

12. **干预和纠错能力**
    - 任务可叫停、取消、恢复
    - 及时发现问题并预警
    - 支持人工介入调整

---

### 优先级建议

- **基础必备**：记忆连续性、沟通规范、权限边界
- **核心能力**：审核把控、数据分析、风险识别
- **进阶能力**：主动学习、进度可视化、知识沉淀
