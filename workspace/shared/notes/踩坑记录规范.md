# 小皮团队踩坑记录规范 v1.0

> 记录各小皮在使用工具、API、技术时遇到的坑，避免重复踩坑，形成团队知识积累。

---

## 📋 规范要求

### 1. 记录时机
- 发现一个坑时，立即记录到 `shared/notes/踩坑记录-{主题}-{日期}.md`
- 发现重复踩坑时，更新已有记录

### 2. 记录格式
```markdown
# {技术/工具}踩坑记录

## 坑N：{简明描述}

**发现时间**：YYYY-MM-DD
**发现小皮**：{小皮名称}
**严重程度**：🔴高 / ⚠️中 / 🟡低

### 问题描述
{详细描述}

### 原因
{分析原因}

### 解决方案
{如何解决}

### 相关API/工具
{相关链接或参考资料}
```

### 3. 管家职责
- 定期收集各小皮的踩坑记录
- 整理归纳到 `shared/notes/踩坑记录汇总.md`
- 重大踩坑及时通知各小皮
