# openclaw.json 配置管理规范

> **皮休强调的重要规范 | 所有小皮必须严格遵守**

---

## 一、文件备份规则

### 1.1 每日自动备份
- **执行时间**：每晚 12:00
- **备份内容**：openclaw.json
- **备份位置**：`~/.openclaw/backups/daily/`
- **命名格式**：`openclaw-YYYY-MM-DD.json`

### 1.2 修改前单独备份
- 每次修改 openclaw.json 前，必须先单独备份
- 备份命名：`openclaw-YYYY-MM-DD-HHMMSS-before-modify.json`
- 确保可追溯和快速恢复

---

## 二、修改申请规则（重要！）

### 2.1 修改前申请
- **必须向皮休申请**，说明：
  - 谁要做修改
  - 要做哪些更新
- **必须等皮休回复"同意"** 才能执行
- 回复任何其他信息都不能修改

### 2.2 申请格式
```
申请修改 openclaw.json
- 申请人：xxx小皮
- 修改内容：xxx
- 原因：xxx
```

---

## 三、Gateway 重启规则

### 3.1 重启前申请
- **必须向皮休申请**，说明：
  - 谁申请重启
  - 做了哪些更新
  - 为什么需要重启
- **必须等皮休回复"同意"** 才能重启
- 回复任何其他信息都不能重启

### 3.2 自动重启
- 皮休回复"同意"后，等待 5 秒钟
- 自动执行重启（无需手动重启）

---

## 四、意外中断处理

### 4.1 自动监测
- 监测 gateway 状态
- 遇断网、断连等意外中断
- 自动重启 gateway

### 4.2 自动重启
- 意外中断后自动监测
- 自动执行重启
- 不要让皮休手动重启

### 4.3 原因汇报
- 重启完成后，第一时间汇报
- 说明是什么导致的重启
- 方便排查问题

---

## 五、重启后任务恢复

### 5.1 立即回忆
- 重启完成后，第一时间回忆
- 检查正在进行的任务
- 确认是否有未完成的工作

### 5.2 向皮休汇报
- 主动向皮休汇报：
  - 是否有未完成的任务
  - 任务当前状态
  - 需要什么支持

---

## 六、反思与改进

### 6.1 每次异常都要反思
- 为什么会出现这样的问题？
- 如何能够避免这种事情的发生？

### 6.2 避免反复出现
- 如果已经出现一次，为什么出现第二次、第三次？
- 反复出现要高度重视
- 制定方案彻底解决

### 6.3 记录到 memory
- 将异常情况和反思记录到 memory
- 积累经验，避免重复犯错
