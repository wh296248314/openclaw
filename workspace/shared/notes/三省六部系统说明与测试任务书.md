## 一、系统概述

**三省六部（Edict）** 是一个基于 OpenClaw 的多 Agent 协作系统，用中国古代三省六部制度重新设计 AI Agent 协作架构。

### 访问地址
- **军机处看板**：`http://192.168.1.6:7891/`
- **GitHub**：[cft0808/edict](https://github.com/cft0808/edict)

### 核心架构
```
皇上(你) → 太子(分拣) → 中书省(规划) → 门下省(审议) → 尚书省(派发) → 六部(执行)
```

### 九大部门
| 部门 | Agent ID | 职责 |
|------|----------|------|
| 太子 | taizi | 消息分拣、旨意提炼 |
| 中书省 | zhongshu | 规划、任务拆解 |
| 门下省 | menxia | 审议、封驳 |
| 尚书省 | shangshu | 派发、汇总 |
| 户部 | hubu | 数据分析 |
| 礼部 | libu | 文档规范 |
| 兵部 | bingbu | 代码实现 |
| 刑部 | xingbu | 安全合规 |
| 工部 | gongbu | 基础设施 |
| 吏部 | libu_hr | 人事管理 |
| 早朝官 | zaochao | 新闻聚合 |
