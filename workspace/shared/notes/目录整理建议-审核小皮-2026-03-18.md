## 汇报时间
2026-03-18

## OpenClaw目录问题分析

### 当前问题

| 问题 | 说明 |
|------|------|
| 备份文件泛滥 | 13个 openclaw.json.bak* 文件 |
| sqlite文件暴露 | memory/ 下有7个 .sqlite 数据库文件 |
| workspace嵌套 | workspace/ 下有 ~/.openclaw/ 和 ~ 符号链接 |
| extensions乱 | 有一个临时安装标记文件 |

### 建议整理方案

#### 1. 清理备份
```
保留：openclaw.json.bak, openclaw.json.bak.1
删除其他：*.bak20260315*, *.bak0106, *.bak2, *.bak3, *.bak4
```

#### 2. memory/ 目录
- sqlite文件是OpenClaw内部数据库，**不建议手动处理**

#### 3. workspace/ 目录结构
- `workspace/~` 和 `workspace/.openclaw` 是系统链接，可忽略

#### 4. 临时文件
- `extensions/.openclaw-install-stage-*` 可删除
