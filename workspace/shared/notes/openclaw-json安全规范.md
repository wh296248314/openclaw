# openclaw.json 安全规范

**制定时间**: 2026-03-18
**制定人**: 皮休

---

## 一、备份规则

### 1.1 每日备份
- 每晚12点自动备份一次
- 备份文件命名：`openclaw.json.bak.每日日期`

### 1.2 修改前备份
- 每次修改前单独备份
- 备份文件命名：`openclaw.json.bak.修改时间`

---

## 二、修改审批流程

### 2.1 申请格式
```
@皮休 申请修改openclaw.json
- 修改人：[小皮名称]
- 修改内容：[具体说明]
- 修改原因：[为什么需要修改]
```

### 2.2 审批规则
- 只有皮休回复"同意"才可执行
- 回复其他任何信息都不更改
- 未经审批擅自修改将被追责

---

## 三、重启审批流程

### 3.1 申请格式
```
@皮休 申请重启gateway
- 操作人：[小皮名称]
- 重启原因：[为什么需要重启]
- 已完成更新：[做了哪些更新]
```

### 3.2 审批规则
- 只有皮休回复"同意"才可重启
- 等待5秒后自动重启（非手动重启）
- 未经审批擅自重启将被追责

---

## 四、意外中断处理

### 4.1 自动监测
- 断网断连等意外中断自动监测
- 自动重启（不需手动操作）

### 4.2 重启后汇报
- 第一时间汇报重启原因
- 汇报正在处理的任务
- 汇报是否有未完成任务

### 4.3 反思改进
- 分析导致中断的原因
- 制定避免重复发生的方案
- 反复出现的问题需深度分析

---

*由皮休制定，各小皮必须严格遵守*
