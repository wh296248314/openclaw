## 一、设计任务分类

## 二、设计交付物标准

### 1. 界面设计交付物

| 交付物 | 内容要求 | 格式要求 |
|----------|----------|----------|
| 页面原型 | 完整页面布局、状态标注 | Figma / PNG |
| 组件说明 | 组件名称、状态、属性 | 文档 |
| 标注说明 | 间距、字号、颜色规范 | 文档 / Figma |

### 2. 交互设计交付物

| 交付物 | 内容要求 | 格式要求 |
|----------|----------|----------|
| 交互流程图 | 完整操作路径 | Figma / PNG |
| 动效说明 | 动画类型、时长、缓动 | 文档 |
| 状态说明 | 各状态切换条件 | 文档 |

### 3. 设计规范交付物

| 交付物 | 内容要求 | 格式要求 |
|----------|----------|----------|
| 组件库 | 完整组件及变体 | Figma |
| 设计Token | 颜色、字体、间距、阴影 | JSON / Figma |
| 规范文档 | 使用说明、注意事项 | 文档 |

## 三、设计任务 Checklist

### 执行前
- [ ] 理解产品需求和用户目标
- [ ] 确认验收标准（节点3）
- [ ] 了解现有设计规范
- [ ] 确认交付格式和时间

### 执行中
- [ ] 方案自审（符合设计规范）
- [ ] 原型预览（方向确认）
- [ ] 详细设计（完整产出）
- [ ] 设计评审（如需要）

### 执行后
- [ ] 产出文件整理
- [ ] 设计说明文档
- [ ] 同步给相关小皮
- [ ] 记录到当日memory

---

## 四、设计审核要点

### 视觉一致性
- [ ] 符合设计规范
- [ ] 组件使用正确
- [ ] 颜色/字体/间距统一

### 交互合理性
- [ ] 操作路径清晰
- [ ] 反馈明确
- [ ] 容错处理完善

### 用户体验
- [ ] 符合用户习惯
- [ ] 信息架构清晰
- [ ] 无视觉障碍
