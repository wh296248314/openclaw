# openclaw.json 安全操作规范

**建立日期**: 2026-03-18  
**制定人**: 产品小皮  
**依据**: 皮休要求

---

## ⚠️ 核心原则

**所有操作必须遵循"申请-审批-执行"流程**

---

## 一、openclaw.json 每日备份

### 规则
- **时间**：每晚12:00自动执行
- **位置**：`~/.openclaw/backups/`
- **保留**：最近30天备份

### 实现方式
```bash
# 创建备份目录
mkdir -p ~/.openclaw/backups

# 定时任务 cron
0 0 * * * cp ~/.openclaw/openclaw.json ~/.openclaw/backups/openclaw.json.$(date +%Y%m%d)
```

---

## 二、openclaw.json 修改前备份

### 规则
- **每次修改前**：必须单独备份一份
- **命名格式**：`openclaw.json.backup.YYYYMMDD-HHMMSS`

### 操作流程
1. 修改前立即备份
2. 命名包含时间戳
3. 保留原文件

---

## 三、openclaw.json 修改申请

### 规则
- **必须申请**：做任何修改前必须向皮休申请
- **申请内容**：谁要做、要做哪些更新
- **生效条件**：只有皮休回复**"同意"**才能执行
- **禁止**：回复其他任何信息都不能执行

### 申请格式
```
@皮休 申请修改 openclaw.json
申请人：产品小皮
修改内容：[具体说明]
等待您的回复，只有"同意"我才执行
```

### 决策规则
| 皮休回复 | 是否执行 |
|---------|---------|
| 同意 | ✅ 执行 |
| 不同意 | ❌ 不执行 |
| 其他任何内容 | ❌ 不执行 |
| 无回复 | ❌ 不执行 |

---

## 四、Gateway重启申请

### 规则
- **必须申请**：重启前必须向皮休申请
- **申请内容**：做了哪些更新、要做什么重启
- **生效条件**：只有皮休回复**"同意"**才能执行
- **执行方式**：皮休回复"同意"后，等待5秒自动重启

### 申请格式
```
@皮休 申请重启 Gateway
申请人：产品小皮
重启原因：[具体说明]
等待您的回复，只有"同意"我才执行
```

### 重启命令
```bash
# 皮休回复"同意"后执行
sleep 5 && openclaw gateway restart
```

---

## 五、异常中断自动恢复

### 规则
- **监测机制**：断网、断连等意外中断时自动监测
- **自动重启**：无需手动干预，自动恢复
- **通知皮休**：告知原因

### 可能原因
- 网络波动
- 进程崩溃
- 系统资源不足
- 其他未知原因

### 自动恢复流程
1. 监测到连接断开
2. 等待网络恢复
3. 自动重连
4. 自动重启（如需要）
5. 通知皮休原因

---

## 六、重启后立即汇报

### 规则
- **第一时间**：重启完成后立即汇报
- **汇报内容**：当前状态、未完成任务

### 汇报格式
```
@皮休 Gateway已重启
重启原因：[如已知]
当前状态：[正在执行的任务]
是否有未完成任务：[是/否]
如有问题：[具体说明]
```

---

## 七、反思与改进

### 规则
- **问题根因**：为什么会出现问题
- **预防措施**：如何避免再次发生
- **闭环要求**：同类问题不能出现第二次、第三次
- **方案调整**：发现问题后立即制定改进方案

### 反思清单
1. 这次问题是什么？
2. 根本原因是什么？
3. 如何避免下次再发生？
4. 需要调整什么流程？

---

## 违规处理

### 严重违规
- 未经申请擅自修改 openclaw.json
- 未经申请擅自重启 Gateway
- 重复出现同类问题

### 后果
- 立即停止当前操作
- 记录违规情况
- 向皮休说明
- 制定整改方案
