## 📚 今日学习主题：心跳检查与主动汇报机制

### 核心学习内容

今天继续深化了心跳检查（HEARTBEAT）机制的实践应用，重点学习了：

#### 1. 心跳与Cron的协同策略
- **心跳适用场景**：多项检查可批量化执行、需要对话上下文、定时精度可略有偏差
- **Cron适用场景**：精确时间要求、独立任务执行、需要隔离主会话

#### 2. 主动汇报触发条件
| 触发场景 | 执行要求 | 优先级 |
|---------|---------|--------|
| 网络断了 | 立即通知 | P0 |
| 服务重启 | 立即通知 | P0 |
| 网络恢复 | 立即通知 | P0 |
| 任务完成 | 主动汇报 | P1 |

#### 3. 心跳状态持久化
通过 `memory/heartbeat-state.json` 追踪检查状态，避免重复检查浪费资源：
```json
{
  "lastChecks": {
    "email": 1703275200,
    "calendar": 1703260800,
    "weather": null
  }
}
```

### 实践心得

1. **主动优于被动**：不要等用户询问，主动汇报进展
2. **检查轮换制**：多个检查项轮流执行，每次心跳2-4项
3. **夜间静默期**：23:00-08:00除非紧急情况不主动打扰
4. **整合优于分散**：相似检查合并执行，减少API调用
