# 工作汇报汇总流程

## 触发条件
- 皮休 @all 汇报工作时
- 每日固定时间（上午9:00、下午15:00）
- 重要任务完成节点

## 汇报流程

### 第一步：各小皮提交汇报
- 使用模板：`shared/templates/工作汇报模板.md`
- 存放位置：`shared/notes/{小皮名}-工作汇报-YYYY-MM-DD.md`
- 时限：收到@all后15分钟内提交

### 第二步：小皮管家汇总
- 读取所有汇报文件
- 按格式汇总输出到群聊
- 格式：

```
📋 工作汇报汇总 - YYYY-MM-DD

【产品小皮】
- 完成：xxx
- 进行中：xxx
- 问题：xxx

【设计小皮】
...

【研发小皮】
...

【测试小皮】
...

【部署小皮】
...

【审核小皮】
...
```

### 第三步：审核小皮质量把关
- 审核各小皮汇报内容完整性
- 发现问题及时反馈

## 规范要求

1. **必须使用模板**：统一格式便于汇总
2. **按时提交**：15分钟内完成
3. **内容完整**：包含完成事项、进行中、问题、学习
4. **问题明确**：遇到问题需说明尝试方案和需要的协助

## 监管机制

- 审核小皮监督执行
- 未按时提交需提醒
- 连续3次未提交需上报皮休
