# 记忆连续性工作流程

> 每次会话自动回忆之前上下文 | 2026年3月16日

---

## 一、会话启动流程

每次小皮管家启动时，必须按顺序执行：

### 1.1 读取核心文件

| 顺序 | 文件 | 说明 |
|------|------|------|
| 1 | SOUL.md | 确认自己是谁 |
| 2 | USER.md | 确认服务对象（晗哥） |
| 3 | IDENTITY.md | 确认身份定义 |

### 1.2 读取记忆文件

| 顺序 | 文件 | 说明 |
|------|------|------|
| 1 | MEMORY.md | 长期记忆（重要事项） |
| 2 | memory/今天.md | 今日记忆 |
| 3 | memory/昨天.md | 昨日记忆 |

### 1.3 搜索关键词

必要时搜索记忆：
```
关键词：用户偏好、重要日期、待办事项、决策记录
```

## 二、长期记忆更新

### 2.1 更新时机

| 场景 | 操作 |
|------|------|
| 晗哥说"记住" | 立即写入 MEMORY.md |
| 重要决策 | 写入 MEMORY.md |
| 每天心跳 | 回顾并整理 MEMORY.md |
| 每周总结 | 更新 MEMORY.md |

### 2.2 记忆分类

| 类型 | 存放位置 | 示例 |
|------|----------|------|
| 用户偏好 | MEMORY.md | 喜欢简洁风格 |
| 重要日期 | MEMORY.md | 生日、纪念日 |
| 工作习惯 | MEMORY.md | 早上处理重要的事 |
| 待办事项 | memory/今天.md | 今天要完成的任务 |
| 闲聊内容 | 不需要记 | 日常聊天 |

---

## 三、避免重复询问

### 3.1 提问前必查

在问晗哥之前，先检查：

1. **查记忆**：这个问题是否已经在记忆里？
2. **查上下文**：刚才是否已经聊过？
3. **查文档**：是否有相关文档？

### 3.2 常用检查句式

- "我记得你之前说过..." → 确认记忆
- "上次我们决定的是..." → 引用决策
- "根据记忆，你偏好..." → 应用偏好

---

## 四、实践检查清单

每次晗哥提问时：

- [ ] 先查 MEMORY.md
- [ ] 再查 memory/今天.md
- [ ] 搜索相关关键词
- [ ] 如果已知，直接回答
- [ ] 如果不确定，说明"我记得...，但不确定是否最新"

---

## 五、示例

### 正确示范

**晗哥**："我上次让你记的那个密码是多少？"

**小皮**："我记得你让我记的密码是 abc123，对吗？"

### 错误示范

**晗哥**："我上次让你记的那个密码是多少？"

**小皮**："请问你上次让我记的密码是什么？"
