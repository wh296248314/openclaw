# 文件命名与目录规范

> 共享空间文件命名规则 | 由小皮维护

---

## 一、文件命名格式

### 通用格式
```
[话题]-[类型]-[名称].md
```

### 示例
| 类型 | 示例 |
|------|------|
| 团队协作 | `三省六部制-小皮团队协作.md` |
| 技术方案 | `AI-技术-多Agent协作方案.md` |
| 需求文档 | `SCRM-需求-用户登录功能.md` |
| 设计稿 | `SCRM-设计-登录页面UI.md` |
| 测试用例 | `SCRM-测试-登录功能用例.md` |
| 部署文档 | `SCRM-部署-Docker配置.md` |

---

## 二、前缀规范

### 按小皮分类

| 小皮 | 文件前缀 | 适用场景 |
|------|----------|----------|
| 主空间 | 无前缀 | 公共文档 |
| 产品小皮 | `产品-` | 需求文档、PRD |
| 设计小皮 | `设计-` | 设计稿、UI规范 |
| 研发小皮 | `研发-` | 技术方案、代码规范 |
| 测试小皮 | `测试-` | 测试用例、缺陷报告 |
| 部署小皮 | `部署-` | 部署文档、运维手册 |

### 按话题分类

| 话题 | 前缀示例 |
|------|----------|
| 项目名 | `SCRM-`、`AI-`、`Web-` |
| 主题 | `三省六部制-`、`工作流-` |
| 类型 | `-需求`、`-设计`、`-技术`、`-测试`、`-部署` |

## 三、目录结构

```
shared/
├── templates/          ← 通用模板
│   ├── 产品-需求模板.md
│   ├── 测试-用例模板.md
│   └── 部署-检查清单.md
├── assets/            ← 静态资源
│   ├── images/
│   └── icons/
├── data/              ← 共享数据
│   ├── dev-API数据.json
│   └── test-测试数据.csv
└── notes/             ← 讨论笔记
    ├── 三省六部制-小皮团队协作.md
    ├── 产品-需求评审记录.md
    └── 研发-技术选型讨论.md
```

---

## 四、命名原则

1. **简洁明确**：名称能看出内容
2. **使用中文**：方便阅读
3. **横杠分隔**：用 `-` 连接词组
4. **避免特殊字符**：只用中文、英文、数字、横杠
5. **时间戳可选**：如需版本控制，加日期 `v2026-03-16`
