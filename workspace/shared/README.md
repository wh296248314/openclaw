# 共享空间文件管理规范

> 各专家小皮在 shared/ 目录下的操作规范

## 目录结构

```
shared/
├── templates/    ← 通用模板（所有人可读可引用）
├── assets/       ← 静态资源（图片、图标等）
├── data/         ← 共享数据（JSON、CSV等）
├── notes/        ← 公共笔记
│   ├── product-xiaopi/
│   ├── design-xiaopi/
│   ├── development-xiaopi/
│   ├── testing-xiaopi/
│   └── deployment-xiaopi/
```

## 文件命名规范

**每个人只能操作自己前缀的文件**：

| 小皮 | 文件前缀 | 示例 |
|------|----------|------|
| 产品小皮 | `product-` | `product-需求模板.md` |
| 设计小皮 | `design-` | `design-配色规范.md` |
| 研发小皮 | `dev-` | `dev-API规范.md` |
| 测试小皮 | `test-` | `test-用例模板.md` |
| 部署小皮 | `deploy-` | `deploy-部署清单.md` |

## 操作规则

### ✅ 允许的操作

1. **读取**：所有人均可读取 shared/ 下任何文件
2. **创建**：在对应自己小皮的子目录下创建文件
3. **修改**：只能修改**自己前缀**的文件
4. **删除**：只能删除**自己前缀**的文件

### ❌ 禁止的操作

1. **禁止覆盖**他人前缀的文件
2. **禁止删除**他人前缀的文件
3. **禁止移动**他人前缀的文件

## 同步机制

各小皮在执行任务时：
1. 先从 `shared/` 读取相关模板/数据
2. 完成任务后，将可共享的产出更新到 `shared/`
3. 保持文件前缀规范，方便追溯

## 冲突处理

如果多人需要修改同一文件：
1. 先在 `notes/对应小皮/` 下创建讨论记录
2. 协商后由文件所属小皮负责修改
3. 修改前备份原文件
