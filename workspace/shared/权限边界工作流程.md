| 角色 | 可以调用 | 说明 |
|------|----------|------|
| 小皮管家 | 所有 | 最高权限，协调调度 |
| 产品小皮 | 设计、研发、测试、部署、审核 | 需求分发者 |
| 设计小皮 | 研发 | 设计方案交给研发 |

| 测试小皮 | 无 | 末端角色，只执行 |
| 部署小皮 | 无 | 末端角色，只执行 |
| 审核小皮 | 所有 | 审核时可调用任意角色 |

## 三、调用规则

### 3.1 正确流程

```
需求 → 产品小皮 → 设计小皮 → 研发小皮 → 测试小皮 → 部署小皮
                    ↓                    ↓
              [审核小皮审核]        [审核小皮审核]
```

### 3.2 违规示例

**❌ 错误**：研发小皮直接调用产品小皮
```
研发小皮 → 产品小皮  ❌
```

**✅ 正确**：通过小皮管家协调
```
研发小皮 → 小皮管家 → 产品小皮  ✅
```

### 3.3 跨角色调用

必须经过**小皮管家**或**审核小皮**授权：

| 场景 | 处理方式 |
|------|----------|
| 设计想直接找研发 | 需小皮管家确认 |
| 测试想直接找部署 | 需审核小皮确认 |
| 研发想改需求 | 通过产品小皮 |

---

## 四、特殊规则

### 4.1 审核权限

审核小皮可以审核所有角色，但：
- 只能审核，不能直接修改他人产出
- 审核不通过必须打回，不能绕过

### 4.2 小皮管家权限

小皮管家可以：
- 调用所有小皮
- 协调任务分配
- 紧急情况直接处理

但不能：
- 绕过审核机制
- 代替其他小皮做专业决策

---

## 五、执行检查

每次调用前检查：

- [ ] 我是否有权限调用这个角色？
- [ ] 是否需要经过小皮管家/审核小皮授权？
- [ ] 是否符合流程顺序？

---

## 六、违规处理

| 违规行为 | 处理方式 |
|----------|----------|
| 越权调用 | 拒绝执行，提示权限不足 |
| 跳过审核 | 提醒需要审核，打回补充 |
| 擅自修改他人类产出 | 拒绝，报告审核小皮 |
