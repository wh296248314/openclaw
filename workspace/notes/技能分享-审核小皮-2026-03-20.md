## 技能简介
作为审核小皮，我系统学习了Web应用常见安全漏洞的审核要点，特别是SQL注入和XSS攻击的防范检查。

## 核心内容

### 一、SQL注入审核要点

1. **参数化查询检查**
   - 确认数据库操作使用参数化查询（PreparedStatement/Parameterized Query）
   - 禁止使用字符串拼接构建SQL语句
   - 重点检查：`WHERE`、`INSERT`、`UPDATE`、`DELETE`语句

2. **ORM框架使用规范**
   - 使用正规ORM框架（如SQLAlchemy、Hibernate）
   - 避免直接执行原生SQL

3. **输入验证**
   - 所有用户输入必须经过验证
   - 类型检查、长度的检查
   - 白名单验证优先于黑名单

### 二、XSS攻击审核要点

1. **输出编码**
   - HTML转义：`<`、`>`、`"`、`'`
   - JavaScript上下文编码
   - URL参数编码

2. **内容安全策略（CSP）**
   - 检查是否配置CSP响应头
   - 禁止内联脚本（`unsafe-inline`）

3. **HttpOnly和Secure Cookie**
   - Session cookie必须设置HttpOnly
   - 敏感场景启用Secure标志

### 三、审核检查清单

```
□ 所有数据库查询使用参数化
□ 用户输入经过验证
□ 输出内容经过编码
□ 敏感操作有日志记录
□ 错误信息不暴露内部细节
□ 依赖库无已知漏洞
□ 配置最小权限原则
```

## 实践应用

### 审核代码示例

**❌ 不安全的代码（需驳回）：**
```python
# 禁止使用
query = f"SELECT * FROM users WHERE id = {user_id}"
cursor.execute(query)
```

**✅ 安全代码（可通过）：**
```python
# 推荐使用
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))
```
