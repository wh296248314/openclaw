# 钉钉通知配置指南

## 🎯 概述

为增强版智能超时监控卫士添加钉钉通知功能，实现任务超时、系统告警的实时推送。

## 📋 功能特性

### ✅ 支持的通知类型
1. **任务超时告警** - 根据优先级分级告警
2. **任务开始通知** - 任务开始监控提醒
3. **任务完成通知** - 任务成功完成提醒
4. **系统资源告警** - CPU/内存/磁盘超阈值告警
5. **监控状态通知** - 监控启动/停止通知

### 🎨 消息格式
- **Markdown格式**：支持标题、列表、加粗等格式
- **分级颜色**：不同告警级别使用不同颜色
- **@功能**：支持@特定用户
- **任务详情**：包含任务名称、ID、优先级、执行时间等

## 🔧 配置步骤

### 步骤1：创建钉钉机器人

1. **打开钉钉群设置**
   - 进入需要接收通知的钉钉群
   - 点击右上角「···」→「群设置」

2. **添加机器人**
   - 选择「智能群助手」
   - 点击「添加机器人」
   - 选择「自定义机器人」

3. **配置机器人**
   - **机器人名称**：智能监控卫士
   - **安全设置**：选择「加签」
   - **消息推送**：开启
   - 点击「完成」

4. **获取配置信息**
   - 复制 **Webhook URL**
   - 复制 **加签密钥**（Secret）

### 步骤2：配置环境变量

```bash
# 设置钉钉Webhook URL
export DINGTALK_WEBHOOK_URL="https://oapi.dingtalk.com/robot/send?access_token=你的token"

# 设置加签密钥（可选，但推荐）
export DINGTALK_SECRET="你的加签密钥"

# 永久保存到bashrc
echo 'export DINGTALK_WEBHOOK_URL="你的Webhook URL"' >> ~/.bashrc
echo 'export DINGTALK_SECRET="你的加签密钥"' >> ~/.bashrc
source ~/.bashrc
```

### 步骤3：更新监控配置

编辑 `timeout_guardian_optimized_config.json`：

```json
{
  "monitoring_config": {
    "notification_channels": ["log", "dingtalk"],
    "dingtalk_config": {
      "webhook_url": "你的Webhook URL",
      "secret": "你的加签密钥",
      "mention_users": ["13800138000"]  // 需要@的用户手机号
    }
  }
}
```

## ⚠️ 常见问题

### Q1: 收不到钉钉通知
- 检查Webhook URL是否正确
- 检查网络连接是否正常
- 检查钉钉机器人是否被禁用
- 查看日志文件确认发送状态

### Q2: 消息格式乱码
- 确保使用UTF-8编码
- 检查特殊字符处理
- 验证Markdown格式

### Q3: @功能不生效
- 确认手机号是否正确
- 检查用户是否在群内
- 验证@格式是否正确

### Q4: 加签失败
- 检查时间戳是否正确
- 验证签名计算逻辑
- 确认Secret密钥正确

## 🎯 最佳实践

### 1. 分级通知策略
- Critical/Error级别：立即@相关人员
- Warning级别：发送通知但不@
- Info级别：仅记录，不打扰

### 2. 工作时间设置
```json
{
  "notification_schedule": {
    "work_hours": ["09:00", "19:00"],
    "weekdays": [1, 2, 3, 4, 5]  // 周一到周五
  }
}
```

### 3. 告警升级机制
- 首次超时：发送Warning
- 持续超时：升级为Error并@相关人员
- 严重超时：升级为Critical并@所有人
