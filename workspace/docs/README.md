# 🛡️ 智能超时监控卫士

一个智能的任务超时监控系统，自动检测任务执行时间并在超时时提醒，确保任务按时完成。

## ✨ 功能特性

### 🎯 核心功能
- **自动超时检测**: 60秒超时阈值自动检测
- **智能提醒**: 最多3次渐进式提醒（第1次警告 → 第2次提醒 → 第3次紧急）
- **实时监控**: 30秒检查间隔，实时掌握任务状态
- **历史记录**: 自动保存任务历史，支持统计分析

### 📊 监控能力
- **任务状态跟踪**: 运行中、已完成、超时
- **耗时统计**: 平均耗时、超时率分析
- **仪表板展示**: 实时可视化监控界面
- **智能预测**: 基于历史数据预测任务完成时间

### 🔧 配置灵活
- 可调整超时阈值（默认60秒）
- 可设置最大提醒次数（默认3次）
- 支持多种提醒方式
- 可自定义任务分类

## 🚀 快速开始

### 方法1: 使用启动脚本（推荐）
```bash
# 给脚本添加执行权限
chmod +x start_guardian.sh

# 运行启动脚本
./start_guardian.sh
```

### 方法2: 直接使用命令行界面
```bash
# 启动监控
python3 guardian_cli.py start

# 查看统计信息
python3 guardian_cli.py stats

# 创建新任务
python3 guardian_cli.py task

# 查看帮助
python3 guardian_cli.py help
```

### 方法3: 直接运行监控核心
```bash
python3 timeout_guardian.py
```

## 📖 使用指南

### 监控界面操作
在监控模式下，可以使用以下命令：
- `s` - 显示状态仪表板
- `t` - 创建测试任务
- `q` - 退出监控
- `h` - 显示帮助

### 任务管理
1. **自动监控**: 系统会自动监控所有启动的任务
2. **手动创建**: 使用 `guardian_cli.py task` 创建自定义任务
3. **状态更新**: 任务状态会自动更新，超时会触发提醒
4. **完成标记**: 任务完成后会自动记录耗时和结果

### 提醒机制
1. **第1次提醒** (超时60秒): 🚨 警告提醒，包含当前状态和预计完成时间
2. **第2次提醒** (继续超时): ⚠️ 加强提醒，要求确认进度
3. **第3次提醒** (继续超时): 🔴 紧急提醒，需要立即关注

## ⚙️ 配置说明

### 配置文件
编辑 `guardian_config.json` 可以自定义设置：

```json
{
  "监控设置": {
    "超时阈值_秒": 60,      // 超时时间阈值
    "最大提醒次数": 3,       // 最多提醒次数
    "检查间隔_秒": 30,       // 检查频率
    "启用桌面通知": true     // 是否显示桌面通知
  }
}
```

### 集成选项
- ✅ **与Obsidian同步**: 自动同步任务记录到Obsidian
- 🔄 **与Git集成**: 监控Git操作耗时
- 📅 **与日历同步**: 基于日历事件设置任务
- 🛠️ **与任务管理工具集成**: 支持第三方工具集成

## 🔧 技术架构

### 核心组件
1. **TaskMonitor**: 任务监控器，负责跟踪任务状态
2. **TimeoutGuardian**: 监控卫士主类，协调监控流程
3. **CLI界面**: 命令行交互界面
4. **配置管理**: JSON配置文件管理

### 监控流程
```
任务开始 → 启动监控 → 定期检查 → 超时检测 → 触发提醒 → 任务完成 → 记录历史
```

- 线程安全的任务管理
