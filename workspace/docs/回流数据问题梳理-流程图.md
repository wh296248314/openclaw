## ⚠️ 回流资源存在的BUG

## 🎯 核心诉求与解决方案

### 方案一：剔除干扰项
```mermaid
flowchart TD
    A[BI未跟进资源提醒] --> B{数据筛选}
    B -->|排除| C[已在读学员数据]
    B -->|仅统计| D[真正需要外呼触达的资源]
    D --> E[准确统计]
    E --> F[提升管理效率]
```

### 方案二：规范更名操作
```mermaid
flowchart TD
    A[需要更改活动名称] --> B{操作对象选择}
    B -->|仅对| C[已回流的数据]
    B -->|避免波及| D[未回流数据<br>在读学员数据]
    C --> E[从源头减少统计干扰]
    D --> F[保持数据纯净]
```

## 📋 关键问题总结

### 数据混淆问题
1. **现象**: 修改活动名称时，系统更新所有历史数据
2. **影响**: 已转化/无需跟进资源被错误归类
3. **后果**: 数据准确性下降，运营判断受干扰

### 统计失真问题
1. **现象**: 未跟进资源提醒计入所有历史数据
2. **影响**: 包含已在读学员等不应统计的数据
3. **后果**: 管理效率降低，数据看板失真
