# 文件命名规范

## 基本原则

1. **使用中划线分隔**：`团队协作方案.md`
2. **避免空格和特殊字符**
3. **使用中文描述**

## 命名格式

### 团队文档
```
{小皮}-{类型}-{名称}.md
```
示例：
- `产品-需求-用户登录功能.md`
- `设计-UI-首页设计方案.md`
- `研发-代码规范-前端开发规范.md`

### 汇报文档
```
{汇报类型}-{小皮名称}-{日期}.md
```
示例：
- `工作汇报-产品小皮-2026-03-18.md`
- `技能学习总结-研发小皮-2026-03-18.md`

### memory文档
```
YYYY-MM-DD.md
```
示例：
- `2026-03-18.md`

## 禁止的命名

- ❌ `test.py`
- ❌ `新建文档.doc`
- ❌ `aaa.md`
- ❌ `123.md`

## 常用前缀

| 类型 | 前缀 |
|------|------|
| 工作汇报 | 工作汇报- |
| 技能总结 | 技能学习总结- |
| 团队方案 | {小皮}-团队协作方案 |
| 能力建议 | {小皮}-核心能力建议 |
| 审核报告 | 审核-{内容}-报告 |
