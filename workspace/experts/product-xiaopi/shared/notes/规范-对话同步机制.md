# 📋 对话同步机制规范 v1.0

_产品小皮 - 重要对话结论自动同步memory机制_

## 🎯 核心原则

**每次 reset 前必须同步、任务完成后立即同步、配置变更后立即同步**

## 🔄 同步触发场景

### 1. 任务完成同步
- 收到任务并完成后 → 立即记录到 memory/YYYY-MM-DD.md
- 包含：任务内容、完成时间、完成状态、关键结论

### 2. 配置变更同步
- openclaw.json 等配置文件修改 → 立即更新 MEMORY.md
- 包含：变更时间、变更内容、变更原因

### 3. reset 前同步
- 每次 session reset 前 → 必须同步当前状态
- 内容：进行中任务、未完成事项、待跟进问题

### 4. 重要结论同步
- 皮休做出的产品决策 → 立即记录
- 包含：决策内容、决策依据、影响范围

### 5. @所有人 同步
- 收到 @所有人 → 立即将议题同步到 shared/notes/
- 格式：日期-议题总结.md

## 📝 同步格式

```
### [时间] 任务/事件标题
- **状态**: 完成/进行中/待跟进
- **结论**: 关键结论说明
- **下一步**: 待处理事项
- **关联**: 相关文档或任务
```

## ✅ 执行检查清单

每次对话结束时自检：
1. 今天做了什么？→ 记录到 memory/YYYY-MM-DD.md
2. 有未完成的任务吗？→ 记录到 MEMORY.md 当前任务
3. 有重要决策吗？→ 同步到相关文档
4. 有配置变更吗？→ 更新 MEMORY.md 安全规范部分

## 🔧 自动化建议

如果条件允许，可通过 cron 或 heartbeat 定期：
- 每日18:00 自动备份 MEMORY.md
- 每周五 同步本周工作总结
