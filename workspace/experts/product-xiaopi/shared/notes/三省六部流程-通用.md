# 三省六部流程 - 研发小皮必读

> 来源：shared/notes/三省六部制-小皮团队协作.md
> 更新时间：2026-03-21

## 核心流程

```
皇帝(皮休) → 太子(产品小皮) → 中书省(规划) → 门下省(审核) → 尚书省(派发) → 六部(执行) → 回奏
```

## 简化版协作流程

```
产品小皮（需求分析+任务分配）
       ↓
设计小皮 + 研发小皮（并行）
       ↓
测试小皮（测试执行）
       ↓
审核小皮（质量审核 ⭐关键！）
       ↓
部署小皮（上线）
```

## 审核机制（重要！）

```
研发交付 → 测试执行 → 审核小皮审核 → 
    ✅ 通过 → 部署小皮
    ❌ 封驳 → 研发返工 → 再次提交 → 循环直到通过
```

**测试小皮拥有测试执行权，审核小皮拥有质量审核权（封驳权）**

## 角色对应

| 小皮 | 类比三省六部 | 职责 |
|------|-------------|------|
| 产品小皮 | 中书省 | 规划、任务分发 |
| 设计小皮 | 礼部 | 设计输出 |
| 研发小皮 | 兵部 | 开发实现 |
| 测试小皮 | 刑部 | 测试执行 |
| 审核小皮 | 门下省 | 质量审核，可封驳 |
| 部署小皮 | 工部 | 上线交付 |

## 关键原则

1. **制度性审核 > 自由协作** - 先规划、再审核、然后执行、最后回奏
2. **审核小皮有权封驳** - 质量不达标可打回重做
3. **完整审计追踪** - 每个任务都有完整流转记录
