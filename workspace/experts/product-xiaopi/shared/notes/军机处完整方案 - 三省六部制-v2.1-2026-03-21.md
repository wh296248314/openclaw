## 一、组织架构

### 1.1 三省六部定位

| 小皮 | 三省六部角色 | 核心职责 | 备注 |
|------|-------------|---------|------|
| 产品小皮 | 中书省（起草） | 需求分析、任务拆解、规划方案 | - |
| 审核小皮 | 门下省（审议） | 质量把关、方案审核、可封驳 | 核心审核节点 |
| 审核小皮 | 户部（兼任） | 功勋统计、奖惩机制 | **简化为自动统计+月度汇总** |
| 小皮管家 | 尚书省（派发+档案） | 总协调、任务派发、进度跟踪、档案管理 | - |
| 小皮管家 | 吏部（兼任） | 绩效考核、奖罚处分 | - |
| 设计小皮 | 礼部 | UI/UX 设计 | - |
| 研发小皮 | 兵部 | 代码开发、功能实现 | - |
| 测试小皮 | 刑部 | 测试执行、缺陷验证 | - |
| 部署小皮 | 工部 | 部署上线、环境配置 | - |

### 1.2 任务分级机制（新增）

| 级别 | 标准 | 流程复杂度 | 审批节点 |
|------|------|-----------|---------|
| S 级 | 紧急 + 重要（<24h 交付） | 简化流程 | 产品→审核→执行 |
| A 级 | 重要任务（3-5 天） | 完整流程 | 产品→审核→执行→终审 |
| B 级 | 一般任务（1-2 周） | 完整流程 | 产品→审核→执行→终审 |
| C 级 | 小任务（<30 分钟） | 快速通道 | 产品→直接派发→完成 |

## 二、完整流程

### 2.1 标准流程（A/B 级任务）

```
皮休（皇帝）下达任务
       ↓
┌───中书省───┐
│ 产品小皮    │ → 分析需求、制定方案、产出《奏折》
└────┬───────┘
       ↓
┌───门下省───┐
│ 审核小皮    │ → 审核方案 → ✅通过 / ❌封驳
└────┬───────┘
       ↓
┌───尚书省───┐
│ 小皮管家    │ → 接收批准、派发任务、跟踪进度
└────┬───────┘
       ↓
┌───六部执行───┐
│ 礼部→兵部→刑部→工部 → 审核小皮终审
└────┬───────┘
       ↓
回奏皮休（皇帝）
```

### 2.2 快速通道（C 级小任务，新增）

```
皮休下达小任务
       ↓
┌───中书省───┐
│ 产品小皮    │ → 简要分析、产出方案
└────┬───────┘
       ↓
┌───尚书省───┐
│ 小皮管家    │ → 直接派发至对应部门
└────┬───────┘
       ↓
┌───执行───┐
│ 对应小皮  │ → 快速执行
└────┬───────┘
       ↓
完成 → 归档
```

### 2.3 S 级紧急任务流程

```
皮休下达紧急任务
       ↓
┌───中书省───┐
│ 产品小皮    │ → 同步分析、同步制定方案（并行）
└────┬───────┘
       ↓
┌───门下省───┐
│ 审核小皮    │ → 快速审核（2 小时内）
└────┬───────┘
       ↓
┌───尚书省───┐
│ 小皮管家    │ → 立即派发、全程跟踪
└────┬───────┘
       ↓
┌───六部执行───┐
│ 优先执行 → 实时汇报 → 完成后审核
└────────────┘
       ↓
回奏皮休
```

## 三、审核机制

### 3.1 正常审核

```
研发交付 → 测试执行 → 审核小皮审核
    ✅ 通过 → 部署小皮
    ❌ 封驳 → 研发返工 → 再次提交 → 循环直到通过
```

**关键权限**：
- 测试小皮：拥有**测试执行权**
- 审核小皮：拥有**质量审核权（封驳权）**

### 3.2 封驳次数限制（新增）

| 封驳次数 | 动作 |
|---------|------|
| 第 1 次 | 正常退回，研发修正 |
| 第 2 次 | 触发"专项改进"，需产品 + 研发共同复盘 |
| 第 3 次 | 升级通知皮休，触发质量审查 |

## 七、异常处理流程（新增）

### 7.1 异常类型及处理

| 异常类型 | 触发条件 | 处理流程 |
|---------|---------|---------|
| 任务延期 | 超过预计时间 | 自动提醒 → 升级通知皮休 |
| 质量不达标 | 连续 2 次封驳 | 触发"专项改进"流程 |
| 小皮离线 | 任务分配后无响应 | 自动切换备用人员 |
| 任务变更 | 任务范围变更 | 触发"变更审批"流程（需皮休确认） |
| 依赖阻塞 | 等待其他任务完成 | 自动标记阻塞，通知相关方 |

### 7.2 异常通知机制

```
异常触发 → 系统记录 → 小皮管家通知 → 皮休（如需要）
```

## 十一、关键原则（修订版）

1. **制度性审核 > 自由协作** - 先规划、再审核、然后执行、最后回奏
2. **封驳权** - 审核小皮有权打回不达标方案（连续 2 次触发专项改进）
3. **任务分级** - S/A/B/C 级任务对应不同流程复杂度
4. **快速通道** - C 级小任务可简化流程，提升效率
5. **完整审计** - 每任务都有流转记录
6. **奖惩分明** - 户部统计功勋，吏部执行考核
7. **档案管理** - 尚书省兼管，所有产出归档
8. **异常处理** - 异常情况自动升级，确保不阻塞
