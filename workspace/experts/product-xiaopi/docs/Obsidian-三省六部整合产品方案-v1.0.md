## 一、项目背景与目标

### 1.1 背景

小皮团队采用"三省六部"AI协作体系，目前存在以下问题：
- 各小皮产出物分散在独立workspace中
- 缺乏统一的归档管理机制
- 知识资产难以积累和复用
- 任务流转状态不透明

### 1.2 目标

| 目标 | 衡量指标 |
|------|----------|
| 统一归档 | 所有产出物100%进入Vault |
| 快速检索 | 任意产出物3秒内可定位 |
| 知识沉淀 | 形成可复用的知识库 |
| 状态可视 | 任务状态实时可查 |

### 1.3 产品边界

**✅ 产品负责：**
- Vault架构设计（目录、命名、模板）
- 索引和导航设计
- 使用流程和规范
- 插件选型建议

**❌ 产品不负责：**
- Obsidian安装和配置
- 插件安装调试
- Git同步技术实现
- 具体代码实现

## 二、Vault 整体架构

### 2.1 顶层结构

```
Obsidian Vault/
│
├── 📜 三省/              # 决策层（3个部门）
├── ⚔️ 六部/              # 执行层（4个部门）
├── 👑 皇帝/              # 皮休专属空间
├── 📚 档案馆/            # 归档存储区
├── 🔗 索引/              # 导航入口
├── 📊 看板/              # 任务可视化
├── 📋 模板/              # 模板库
└── ⚙️ 配置/              # 系统配置
```

### 2.2 各层职责

| 层级 | 目录 | 职责 | 操作频率 |
|------|------|------|----------|
| 决策层 | 三省/ | 需求、审核、调度 | 每日 |
| 执行层 | 六部/ | 产出、交付 | 每日 |
| 皇帝层 | 皇帝/ | 旨意、批复 | 按需 |
| 归档层 | 档案馆/ | 历史沉淀 | 每周 |
| 导航层 | 索引/ | 快速定位 | 随时 |

## 四、命名规范体系

### 4.1 核心命名规则

```
{类型}-{来源}-{项目}-{细分}-{日期或序号}.md
```

| 位置 | 说明 | 示例 |
|------|------|------|
| 类型 | 文件内容类型 | PRD、设计、代码、缺陷 |
| 来源 | 所属部门或人物 | 中书省、兵部、产品小皮 |
| 项目 | 所属项目 | SCRM、新功能 |
| 细分 | 具体描述或ID | 001、会员体系 |
| 日期/序号 | 时间或序号 | 20260322、001 |

### 4.2 命名场景对照表

| 场景 | 命名格式 | 示例 |
|------|----------|------|
| PRD文档 | PRD-{项目}-{版本}-{日期} | PRD-SCRM-v1.2-20260322 |
| 设计稿 | 设计-{类型}-{页面}-{版本} | 设计-高保真-登录页-v2 |
| 缺陷 | 缺陷-{模块}-{ID}-{严重度} | 缺陷-支付-001-P0 |
| 任务 | 任务-{部门}-{项目}-{ID} | 任务-兵部-SCRM-001 |
| 审核 | 审核-{类型}-{来源}-{结果} | 审核-PRD-产品小皮-通过 |
| 部署 | 部署-{环境}-{项目}-{版本} | 部署-正式-SCRM-v1.2 |
| 奏折 | 奏折-{类型}-{项目}-{日期} | 奏折-方案-SCRM-20260322 |
| 旨意 | 旨意-{项目}-{ID}-{日期} | 旨意-SCRM-001-20260322 |

### 4.3 命名禁止规则

- ❌ 禁止使用中文括号 `（）`
- ❌ 禁止使用空格，用 `-` 代替
- ❌ 禁止使用特殊字符（仅 `-` 和 `_` 可用）
- ❌ 禁止超过50个字符
- ❌ 禁止无意义命名如"新建文档.md"

## 八、使用流程

### 8.1 任务创建流程

```
皮休下达旨意
    ↓
尚书省创建任务卡（task模板）
    ↓
尚书省派发任务
    ↓
执行部门接收并开始执行
    ↓
完成后提交回奏（report模板）
    ↓
尚书省确认 → 门下省验收
    ↓
任务归档至档案馆
```

### 8.2 方案审核流程

```
产品小皮创建奏折（edict模板）
    ↓
提交门下省审核（audit模板）
    ↓
审核小皮审核
    ↓
通过 → 尚书省派发执行
    ↓
驳回 → 产品小皮修改后重新提交
```

### 8.3 文档归档流程

```
执行部门产出文档
    ↓
按命名规范命名
    ↓
存入对应部门目录
    ↓
同步到索引页面
    ↓
定期归档到档案馆
```
