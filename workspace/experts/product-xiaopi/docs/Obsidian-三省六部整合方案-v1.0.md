## 一、项目概述

### 1.1 背景

当前小皮团队采用"三省六部"AI协作体系，存在以下问题：
- 产出物分散在各专家的 workspace 中
- 缺乏统一的档案管理和检索机制
- 知识积累难以形成体系

### 1.2 目标

将 **Obsidian** 作为团队的**统一档案馆 + 知识库**，承载三省六部的所有产出物。

### 1.3 预期价值

| 价值点 | 说明 |
|--------|------|
| 统一归档 | 所有产出物集中管理 |
| 快速检索 | 通过双向链接、标签快速定位 |
| 知识沉淀 | 形成团队知识资产 |
| 可视化看板 | 任务状态一目了然 |

## 三、功能需求

### 3.1 目录结构

```
Obsidian Vault/
├── 三省/              # 决策层产出
│   ├── 中书省/        # 需求、PRD、方案起草
│   ├── 门下省/        # 审核记录
│   └── 尚书省/        # 任务派发、进度
│
├── 六部/              # 执行层产出
│   ├── 礼部/          # 设计文档
│   ├── 兵部/          # 技术文档
│   ├── 刑部/          # 测试报告
│   └── 工部/          # 部署文档
│
├── 皇帝/              # 皮休专属空间（旨意下达）
│
├── 档案馆/            # 归档区（六部产出汇总）
│   ├── 按项目/
│   └── 按时间/
│
└── 索引/              # 导航入口
```

### 3.1.1 业务流程

```
皇帝（皮休）下旨/日报/任务
      ↓
三省决策（中书省起草 → 门下省审核）
      ↓ 无问题后
尚书省派发（根据任务类型）
      ↓
六部执行（产出最终成果）
      ↓
档案馆归档
```

### 3.2 文件命名规范

```
{类型}-{来源部门}-{项目}-{日期}-{序号}.md
```

**示例：**
- `PRD-中书省-SCRM-20260322-001.md`
- `审核-门下省-SCRM-20260322-002.md`
- `设计-礼部-SCRM-20260322-003.md`

### 3.3 核心模板

| 模板 | 用途 | 关键字段 |
|------|------|----------|
| 任务卡 | 任务流转 | from, to, status, priority |
| 奏折 | 方案汇报 | author, date, proposals |
| 审核单 | 审核记录 | target, result, comments |
| 回奏 | 完成汇报 | task_id, result, evidence |

### 3.4 索引体系

| 索引 | 内容 |
|------|------|
| dashboard | 总仪表盘，任务概览 |
| tasks | 任务列表（可筛选） |
| members | 小皮状态一览 |

### 3.5 标签体系

| 标签类型 | 示例 |
|----------|------|
| 部门 | #中书省 #兵部 #工部 |
| 项目 | #SCRM #新功能 |
| 状态 | #待处理 #进行中 #已完成 |
| 优先级 | #P0 #P1 #P2 |

## 四、非功能需求

- 标签体系可灵活调整
