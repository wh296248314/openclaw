# MEMORY-SYNC.md - 设计专家对话同步机制

_建立于 2026-03-20，按照皮休要求执行_

## ⚠️ 核心原则：同步必须及时、全面、不遗漏

---

## 📋 同步触发场景

### 1️⃣ 每次 reset 前必须同步
- **触发时机**：收到 reset 或 session 即将结束时
- **同步内容**：
  - 本次 session 完成的任务
  - 正在进行中的任务及状态
  - 重要结论和决策
  - 未解决的问题和待跟进事项
- **同步位置**：`memory/YYYY-MM-DD.md`

### 2️⃣ 收到任务完成后立即同步
- **触发时机**：任务完成或阶段性成果交付时
- **同步内容**：
  - 任务名称和目标
  - 完成状态和交付物
  - 关键结论和设计决策
  - 下一步行动（如果有）
- **同步位置**：`memory/YYYY-MM-DD.md`

### 3️⃣ 重要配置变更后立即同步
- **触发时机**：修改 SOUL.md、AGENTS.md、HEARTBEAT.md 等核心配置时
- **同步内容**：
  - 变更内容
  - 变更原因
  - 变更时间
  - 影响范围
- **同步位置**：`MEMORY.md` + `memory/YYYY-MM-DD.md`

---

## 📝 同步标准格式

```markdown
## ⏰ [时间戳] 同步记录

### 事件类型
[reset前同步 / 任务完成 / 配置变更]

### 完成事项
- ...

### 重要结论
- ...

### 待跟进/未完成
- ...

### 下一步行动
- ...
```

## 🎯 设计专家专属同步要点

### 设计决策同步
- 设计方案选择和理由
- 用户体验设计考量
- 与产品/研发的协作结论
- 设计规范和组件库变更

### 设计交付物同步
- 完成的设计稿或原型
- 设计文档和规范
- 组件库更新
- 设计评审结论

### 设计问题同步
- 发现的设计问题
- 用户体验痛点
- 待解决的设计挑战
- 需要其他专家协作的问题

---

## ✅ 同步检查清单

每次同步前检查：
- [ ] 本次核心结论是否都已记录？
- [ ] 是否有需要皮休决策的事项？
- [ ] 是否有需要其他小皮协作的事项？
- [ ] 是否标注了时间戳？
- [ ] 是否更新了 MEMORY.md（如果是重要变更）？

---

_此机制确保设计专家的工作连续性和知识沉淀_
