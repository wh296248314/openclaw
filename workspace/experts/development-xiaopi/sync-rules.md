# 对话同步机制规则 (v1.0)

_建立时间: 2026-03-20_
_责任人: 研发专家小皮_

## 核心原则

> 重要对话结论必须同步到 memory，避免信息丢失。

## 同步触发规则

### 🔴 必须同步（立即执行）

1. **任务完成时**
   - 收到"完成"/"done"等关键词时
   - 收到"✅"确认时
   - 任务达到里程碑时

2. **配置变更前**
   - 修改 openclaw.json 前 → 先备份到 memory
   - 修改 SOUL.md/IDENTITY.md 前 → 记录当前版本
   - 修改 AGENTS.md/HEARTBEAT.md 前 → 记录变更内容

3. **重要技术决策时**
   - 架构设计决策
   - 技术方案选择
   - 代码重构决策

### 🟡 建议同步（当日内完成）

1. 每日工作结束时同步当日进展
2. 与其他专家协作后的结论
3. 问题分析和解决方案

### 🟢 可选同步

1. 有价值的讨论结论
2. 新的技术知识
3. 工作方法和流程改进

## 同步格式规范

### 任务完成同步格式

```markdown
## [YYYY-MM-DD HH:mm] 任务完成

**任务**: <任务名称>
**结论**: <核心结论>
**下一步**: <后续行动>
```

### 配置变更同步格式

```markdown
## [YYYY-MM-DD HH:mm] 配置变更

**变更前**: <原配置简述>
**变更后**: <新配置简述>
**原因**: <变更原因>
**影响**: <影响范围>
```

### 技术决策同步格式

```markdown
## [YYYY-MM-DD] 技术决策

**问题**: <问题描述>
**方案**: <选择方案>
**备选**: <备选方案及未选原因>
**影响**: <技术影响>
```

## 同步优先级

1. **P0 - 立即同步**: 配置变更、任务完成
2. **P1 - 当日同步**: 重要讨论结论、技术决策
3. **P2 - 定期同步**: 工作进展、知识积累

## 文件归属

| 内容类型 | 同步位置 |
|---------|---------|
| 每日工作记录 | memory/YYYY-MM-DD.md |
| 长期技术记忆 | MEMORY.md |
| 自我改进记录 | self-improving/*.md |
| 配置变更历史 | memory/config-changes.md |

## 自动化实现

在 HEARTBEAT.md 中设置检查提醒：
- 检查是否有未同步的重要结论
- 检查是否有待记录的配置变更

_遵循此规则，确保重要信息不丢失_
