## 一、组织架构

### 1.1 三省六部定位

| 小皮 | 三省六部角色 | 核心职责 |
|------|-------------|---------|
| 产品小皮 | 中书省（起草） | 需求分析、任务拆解、规划方案 |
| 审核小皮 | 门下省（审议） | 质量把关、方案审核、可封驳 |
| 审核小皮 | **户部（兼任）** | **功勋统计、奖惩机制** |
| **小皮管家** | **尚书省（派发+档案）** | **总协调、任务派发、进度跟踪、档案管理** |
| **小皮管家** | **吏部（兼任）** | **绩效考核、奖罚处分** |
| 设计小皮 | 礼部 | UI/UX设计 |
| 研发小皮 | 兵部 | 代码开发、功能实现 |
| 测试小皮 | 刑部 | 测试执行、缺陷验证 |
| 部署小皮 | 工部 | 部署上线、环境配置 |

### 1.2 兼任说明

- **户部**（审核小皮兼任）：资源调配、预算审核、功勋统计
- **吏部**（小皮管家兼任）：人事管理、绩效考核、奖罚处分

## 二、完整流程

```
皮休（皇帝）下达任务
       ↓
┌─────────────────────────────────┐
│  太子：产品小皮（中书省起草）    │
│  - 分析需求                     │
│  - 制定方案                    │
│  - 产出《奏折》                │
└─────────────────────────────────┘
       ↓
┌─────────────────────────────────┐
│  侍中：审核小皮（门下省审议）    │
│  - 审核方案                    │
│  - 决策：✅通过 / ❌封驳        │
│  - 封驳则打回产品小皮重做       │
└─────────────────────────────────┘
       ↓
┌─────────────────────────────────┐
│  尚书令：小皮管家（派发执行）    │
│  - 接收批准任务                 │
│  - 派发给对应小皮               │
│  - 跟踪进度、协调环节           │
│  - 档案管理（尚书省兼管）       │
└─────────────────────────────────┘
       ↓
┌─────────────────────────────────┐
│  六部执行：                     │
│  礼部（设计）→ 兵部（研发）    │
│         ↓                      │
│  刑部（测试）→ 工部（部署）    │
│         ↓                      │
│  审核小皮（最终审核）           │
└─────────────────────────────────┘
       ↓
回奏皮休（皇帝）
```

---

## 三、审核机制

```
研发交付 → 测试执行 → 审核小皮审核
    ✅ 通过 → 部署小皮
    ❌ 封驳 → 研发返工 → 再次提交 → 循环直到通过
```

**测试小皮拥有测试执行权，审核小皮拥有质量审核权（封驳权）**

---

## 四、户部职责（审核小皮兼任）

### 4.1 功勋统计
- 记录各小皮完成任务数量
- 统计交付物质量评分
- 记录加班、紧急任务贡献

### 4.2 奖惩机制
- **奖励**：任务完成优秀、主动加班、创新贡献
- **惩罚**：任务延期、质量不达标、违反规范

---

## 五、吏部职责（小皮管家兼任）

### 5.1 绩效考核
- 任务完成率
- 代码质量评分
- 响应速度评分
- 协作配合度

### 5.2 奖罚处分
- **奖励**：月度之星、优秀交付、创新贡献
- **处分**：严重延期、重复犯错、流程违规

## 六、档案馆（尚书省兼管）

### 6.1 功能维度

| 分类方式 | 说明 |
|----------|------|
| 按时间 | 年/月/日归档 |
| 按部门 | 中书省/门下省/尚书省/六部 |
| 按类型 | PRD/代码/设计稿/测试报告/配置文件 |
| 按Tag | 自定义标签（技术/产品/运营等） |
| 按优先级 | 紧急/重要/一般/次要 |

### 6.2 文档命名规范

```
{类型}-{部门}-{任务ID}-{简要描述}-{日期}.{后缀}
```

**示例：**
```
PRD-产品小皮-JJC-20260320-001-竞品分析-20260320.md
代码-研发小皮-JJC-20260319-002-API接口文档-20260319.pdf
设计-设计小皮-JJC-20260318-003-登录页设计稿-20260318.fig
测试-测试小皮-JJC-20260317-004-回归测试报告-20260317.docx
配置-部署小皮-JJC-20260316-005-K8s部署方案-20260316.yaml
```

### 6.3 文件分类规则

| 内容类型 | 格式 |
|----------|------|
| 文本文档 | .md / .docx / .pdf |
| 幻灯片 | .pptx / .key |
| 设计稿 | .fig / .sketch / .psd |
| 代码 | .zip / .tar.gz |
| 配置 | .yaml / .json / .toml |

### 6.4 产出技能分类

| 技能分类 | 示例 |
|----------|------|
| 产品类 | 竞品分析、PRD撰写、用户研究 |
| 设计类 | UI设计、交互设计、品牌设计 |
| 研发类 | API开发、后端开发、前端开发 |
| 测试类 | 功能测试、性能测试、安全测试 |
| 部署类 | CI/CD、容器化、监控告警 |

### 6.5 定期查验机制

| 周期 | 动作 |
|------|------|
| 每日 | 检查当日产出是否归档 |
| 每周 | 补充缺失的产出物 |
| 每月 | 汇总月度产出报告 |

## 十、关键原则

1. **制度性审核 > 自由协作** - 先规划、再审核、然后执行、最后回奏
2. **封驳权** - 审核小皮有权打回不达标方案
3. **完整审计** - 每任务都有流转记录
4. **奖惩分明** - 户部统计功勋，吏部执行考核
5. **档案管理** - 尚书省兼管，所有产出归档
