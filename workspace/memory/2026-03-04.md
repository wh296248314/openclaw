

## 🕕 18:03 LifeOS文档格式学习

### 📚 文档格式规范学习
通过重新阅读LifeOS文档，我理解了正确的文档结构：

#### ✅ 正确的LifeOS文档结构
1. **项目列表** - 使用LifeOS插件自动生成：`<% LifeOS.Project.snapshot() %>`
2. **日常记录** - 标准任务列表格式：
   ```
   - [ ] 时间 任务描述 #标签 📅 截止日期
   ```
3. **输出** - 打卡记录（不会被统计为任务）
4. **精力分配** - 使用LifeOS插件：````LifeOS ProjectListByTime````
5. **今日截止** - 使用LifeOS插件：````LifeOS TaskDueListByTime````
6. **今日完成** - 使用LifeOS插件：````LifeOS TaskDoneListByTime````

#### ❌ 我之前犯的错误
1. **添加了自定义表格格式** - 不符合LifeOS标准格式
2. **创建了新的章节** - 如"今日待办事项清单"、"今日工作待办任务"
3. **使用了复杂的表格结构** - 而不是简单的任务列表

#### 📝 正确的任务记录格式示例
从2026-01-22.md中学习到的正确格式：
```
- [x] 17:58 09:00 - 09:30 【待上线】已完成数据修复，已在预发布验证 #SCRM 🔼 📅 2026-01-22 ✅ 2026-01-23
	- 1.商机优化
	- 2.招生工作台，TMK工作台负责人显示为空
	- 3.导入日志增加
```

### 🎯 重要规则总结
1. **不自由发挥** - 严格按照用户定义的文档格式
2. **只在"日常记录"部分添加内容** - 使用标准任务列表格式
3. **使用简单的时间+描述+标签格式** - 不添加复杂表格
4. **等待用户指示** - 当需要调整格式时，按用户要求修改
