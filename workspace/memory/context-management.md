# 智能上下文管理机制

## 设计理念
借鉴LobeChat等先进AI应用框架，实现智能的上下文裁剪、压缩和加载策略，避免上下文超限，保持对话连贯性。

## 核心策略

### 1. 逆序优先保留策略
```
最新消息 → 最近相关对话 → 早期历史 → 压缩/丢弃
```

**实施规则：**
- **优先级1**：最近10条消息（完整保留）
- **优先级2**：最近30条消息（选择性保留关键对话）
- **优先级3**：30条之前的历史（压缩为摘要或选择性丢弃）

**保留逻辑：**
```javascript
// 伪代码逻辑
function prioritizeContext(messages) {
  const total = messages.length;
  if (total <= 15) return messages; // 小对话完整保留
  
  // 逆序处理：从最新开始
  const prioritized = [];
  const recentCount = Math.min(10, total);
  
  // 1. 保留最近10条（完整）
  prioritized.push(...messages.slice(-recentCount));
  
  // 2. 保留关键对话（包含特定关键词）
  const important = messages.slice(0, -recentCount)
    .filter(msg => containsKeywords(msg));
  prioritized.push(...important.slice(-5)); // 最多5条关键历史
  
  // 3. 早期历史压缩
  if (total > 20) {
    const earlyHistory = messages.slice(0, -15);
    const summary = compressToSummary(earlyHistory);
    prioritized.unshift(summary); // 摘要放在最前面
  }
  
  return prioritized;
}
```

### 2. 内容压缩与摘要策略

#### A. 对话历史压缩
**压缩时机：**
- 对话轮次 > 20轮
- 上下文长度 > 80%限制
- 涉及长期项目讨论

**压缩方法：**
```
原始对话（20轮） → 提取主题 → 生成摘要 → 保留关键决策
```

**摘要格式：**
```markdown
📝 **历史摘要** [时间范围]
- **主要话题**：[话题1]、[话题2]、[话题3]
- **关键决策**：[决策1]、[决策2]
- **待办事项**：[任务1]、[任务2]
- **当前状态**：[进行中/已完成]
```

#### B. 文件内容压缩
**压缩规则：**
- 长文档：提取目录结构和关键章节
- 代码文件：保留函数签名和关键逻辑
- 数据文件：保留数据结构和样本

### 3. 文件分块与懒加载策略

#### A. 大文件分块处理
```
完整文件 → 分析结构 → 提取关键块 → 按需加载
```

**分块策略：**
1. **首部加载**：文件开头1KB（了解文件类型和结构）
2. **目录加载**：提取目录/章节结构
3. **关键章节**：用户指定的或自动识别的关键部分
4. **按需扩展**：提供"加载更多"选项

#### B. 懒加载实现
**交互模式：**
```markdown
📄 **文件分析：example.pdf** (2.5MB)
✅ **已加载：**
- 文件类型：PDF文档
- 总页数：50页
- 章节结构：1.引言 2.方法 3.结果 4.讨论

🔍 **关键内容摘要：**
- 第1-2页：研究背景和目的
- 第10-15页：核心方法描述
- 第30-35页：主要实验结果

📥 **可选加载：**
- [加载第3-5页：详细背景]
- [加载第16-20页：实验设计]
- [加载完整讨论部分]
```

### 4. Token使用监控与优化

#### A. 实时监控
```markdown
🧮 **上下文使用情况**
- 当前Token：12,345/16,384 (75%)
- 剩余空间：4,039 Token
- 建议操作：[保持/压缩/清理]
```

#### B. 自动优化策略
**触发条件：**
- Token使用 > 85% → 启动压缩
- Token使用 > 90% → 强制清理早期历史
- 连续对话 > 30轮 → 自动生成摘要

**优化动作：**
1. **轻度优化**：压缩最早5轮对话为摘要
2. **中度优化**：清理不相关的早期历史
3. **重度优化**：只保留最近10轮+关键摘要

### 5. 上下文恢复与回溯

#### A. 上下文快照
**保存时机：**
- 每10轮对话保存一次快照
- 话题切换时保存快照
- 重要决策点保存快照

**快照格式：**
```json
{
  "timestamp": "2026-02-27 18:20",
  "topic": "智能上下文管理机制讨论",
  "key_points": ["逆序保留", "内容压缩", "懒加载"],
  "context_hash": "abc123",
  "file_reference": "memory/context-snapshots/2026-02-27-1820.json"
}
```

#### B. 回溯机制
**用户请求：** "回到刚才讨论的XX话题"
**系统响应：**
1. 查找相关话题的快照
2. 加载快照上下文
3. 恢复对话状态
4. 继续讨论

## 使用示例

### 场景：分析100页技术文档
**传统方式：** 一次性加载 → 可能超限 → 失败
**智能方式：**
1. **步骤1**：加载文档元数据（文件类型、页数、目录）
2. **步骤2**：提取关键章节（摘要、结论、核心方法）
3. **步骤3**：按需加载详细内容（用户指定部分）
4. **步骤4**：生成综合分析报告

**Token节省：** 从可能超限 → 控制在50%以内
