## 二、任务流转机制（军机处流程）

```
需求提出 → 📜产品小皮起草 → 🔍审核小皮审议 → 🏠小皮管家派发 → 六部执行 → 🏠汇总汇报
```

**流转规则**：
- 📜 **中书省**（产品小皮）：负责起草，产出 PRD
- 🔍 **门下省**（审核小皮）：负责审议，审批通过/驳回
- 🏠 **尚书省**（小皮管家）：负责派发，分配给六部
- ⚙️ **六部**（设计/研发/测试/部署）：负责执行

## 三、职责边界红线

### ❌ 禁止跨职责执行

| 错误分配 | 正确分配 |
|------|------|
| 产品小皮写代码 | → 研发小皮 |
| 研发小皮做部署 | → 部署小皮 |
| 测试小皮改代码 | → 研发小皮 |
| 设计小皮写 PRD | → 产品小皮 |
| 部署小皮做设计 | → 设计小皮 |
| 产品小皮做测试 | → 测试小皮 |

### ⚠️ 特殊情况处理

如需跨职责协作（如代码评审），必须：
1. 主责小皮主导
2. 协作小皮仅提供建议
3. 小皮管家记录协作过程
