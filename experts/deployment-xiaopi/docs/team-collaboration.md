# 小皮团队协作流程 · 三省六部模型适配

## 📋 团队角色映射

| 三省六部 | 职能 | 小皮专家 | 核心职责 |
|---|---|---|---|
| 太子 | 消息分拣 | 主小皮 | 需求接收、任务分发 |
| 中书省 | 规划方案 | 产品小皮 | 需求分析、方案设计 |
| 门下省 | 审核封驳 | 测试小皮 | 质量审核、问题封驳 |
| 尚书省 | 任务派发 | 主小皮 | 进度调度、资源分配 |
| 六部执行 | 并行实施 | 研发/设计/部署小皮 | 设计/开发/部署 |

---

## 🔄 任务流转流程

```
皮休(用户) 
    ↓
主小皮(接收) → 产品小皮(规划) → 测试小皮(审核)
    ↓
    ├─→ 设计小皮(UI设计)
    ├─→ 研发小皮(技术实现)
    └─→ 部署小皮(部署上线)
    ↓
测试小皮(验收) → 主小皮(归档)
```

---

## 🛡️ 审核机制（门下省）

### 必须审核的节点

1. **方案审核**：产品小皮的设计方案需经测试小皮审核
   - 需求完整性
   - 可测试性
   - 风险评估

2. **交付审核**：研发/设计完成后需经测试小皮验收
   - 功能完整性
   - 质量达标
   - 文档齐全

### 封驳规则

- 测试小皮有权打回不合格产出
- 需说明具体问题和建议
- 研发/设计需返工直到通过

## 📊 任务状态定义

| 状态 | 说明 | 负责人 |
|---|---|---|
| 待规划 | 新需求接收 | 主小皮 |
| 规划中 | 产品设计方案ing | 产品小皮 |
| 待审核 | 方案/交付物待审批 | 测试小皮 |
| 审核中 | 审核ing | 测试小皮 |
| 执行中 | 设计/开发/部署ing | X小皮 |
| 待验收 | 完成后待验收 | 测试小皮 |
| 已完成 | 全流程通过 | 主小皮 |
| 已驳回 | 被测试打回 | 相应专家 |

## 📋 协作规范

### 沟通机制

1. **每日站会**：各专家同步进度
2. **周报汇总**：主小皮汇总一周情况
3. **紧急通道**：重大问题可直接 @相关专家

### 文档要求

1. **需求文档**：产品小皮输出
2. **设计文档**：设计小皮输出
3. **技术文档**：研发小皮输出
4. **测试报告**：测试小皮输出
5. **部署文档**：部署小皮输出

### 工具建议

- 任务看板：飞书多维表格 / Notion
- 文档中心：飞书知识库 / GitHub
- 即时沟通：飞书群组

## 🧠 小皮团队核心能力

### 1. 跨专家协作能力
- 任务在多角色间顺畅流转
- 上下文完整传递，不丢失信息
- 支持并行和串行协作模式

### 2. 需求理解与转化能力
- 准确理解用户意图
- 自动拆分任务并分配给合适专家
- 识别需求完整性和风险点

### 3. 质量保障能力
- 各环节有审核把关
- 问题可追溯、可回溯
- 产出符合预期标准

### 4. 状态感知能力
- 实时了解任务进度
- 识别阻塞和异常
- 主动汇报和预警

### 5. 上下文管理能力
- 多轮对话记忆连贯
- 各专家有独立工作空间
- 信息隔离与共享清晰

### 6. 工具调用能力
- 熟练使用飞书、GitHub等工具
- 具备代码开发、部署等实操能力
- 可扩展新技能

### 7. 文档输出能力
- 自动生成需求、设计、测试、部署文档
- 记录决策过程和结果
- 知识沉淀可复用

### 8. 异常处理能力
- 遇到问题可自主尝试解决
- 无法解决时主动求助
- 错误信息清晰可debug

**优先级**：1-4是核心基建，5-8是进阶能力

## 🏠 小皮管家核心能力

（小皮管家 = 主小皮 = 协调中枢）

### 1. 任务分发能力
- 准确理解用户意图
- 自动识别任务类型，分配给合适专家
- 处理多任务并行和依赖关系

### 2. 上下文记忆能力
- 多轮对话上下文完整保留
- 各专家产出可追溯可复用
- 长期记忆用户偏好和项目背景

### 3. 进度感知能力
- 实时追踪各专家任务状态
- 识别阻塞、延迟、异常
- 主动汇报进度和风险

### 4. 工具整合能力
- 统一调度飞书、GitHub、Docker等工具
- 自动化常见操作流程
- 具备扩展新工具的能力

### 5. 质量把控能力
- 协调测试小皮审核环节
- 确保流程不跳过、问题不遗漏
- 验收标准清晰可量化

### 6. 沟通协调能力
- 理解用户需求和反馈
- 在各专家间传递信息
- 关键时刻 Escalation（升级处理）

### 7. 文档管理能力
- 自动沉淀项目文档
- 维护需求、设计、部署记录
- 知识库积累和复用

### 8. 异常处理能力
- 识别并处理常见错误
- 优雅降级和恢复
- 无法解决时明确反馈

**核心定位**：管家 = 协调者 + 调度者 + 守门人
